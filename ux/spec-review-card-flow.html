必须遵守的边界

✓工具/模型选择、返回、移除、清空都 PATCH 原选择卡。
✓`/spec <需求>` 的需求就是目标，确认后直接开始 Spec。
✓review agent 池只影响 REVIEW 阶段多角色审查分配。
✓Auto 不额外选择工具，沿用主任务 agent + 模型。
✕不能展示 Worktree 四步骤、Worktree 标题或等待目标。
✕不能在群里另发一张 Worktree 等待目标卡。

后端事件状态机

输入 `/spec <需求>` 绑定 Spec topic，保存 pending requirement，创建 review-agent 选择卡。
选择工具 `spec_review_select_tool` 写入 pending tool；如果有模型列表，原卡切到模型选择态。
选择模型 `spec_review_select_model` 把工具+模型加入 Spec review selection state，原卡回到工具池态。
确认 `spec_review_finish_selection` 读取 pending requirement 和已选 pool，直接调用 Spec 启动。
执行 `SpecEngine.execute()` 进入 SPEC → PLAN → TASK → BUILD → REVIEW 主循环。
禁止 `worktree_confirm` Spec 不允许进入 WT confirm/awaiting-goal/reply 新卡链路。
