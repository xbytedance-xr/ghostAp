收敛判定与熔断机制
循环终止条件（ContinuationPolicy）
每轮 BUILD+REVIEW 后，引擎评估以下条件决定是否继续迭代
✅
SUCCESS 成功
所有验收标准 PASS + 审查连续通过 ≥2 次（pass_streak_required=2）
🔄
CONVERGED 收敛
连续 N 轮（convergence_window=2）标准满足数和建议集合完全不变 → 死循环检测
📦
BACKLOG_STUCK 积压
待办优化项长期未消减，discovery 发现的问题无法被后续循环解决
💥
CONSECUTIVE_FAILURES
连续 N 个循环异常失败（ACP 超时、模型错误），触发终止保护
🔢
MAX_CYCLES 上限
达到最大轮次（默认 1000，上限 5000）。infinite_mode 可绕过
⏸️
PAUSED 暂停
用户主动 /spec_pause 或 /stop_spec，引擎安全暂停并持久化状态
审查熔断器 (ReviewCircuitBreaker)
防止审查持续失败导致资源浪费，采用指数退避策略
🟢
CLOSED（正常）
每轮正常执行多角色审查
连续失败 N 次 ↓
🔴
OPEN（熔断）
跳过 ACP 审查 → 仅执行轻量 Lint 检查 → 指数退避冷却 → 冷却结束后自动恢复正常审查
