## CircuitBreaker 增强（4项功能扩展）

### 任务描述
增强 `src/utils/circuit_breaker.py`，添加 4 项新功能：async_call 异步调用、reset 强制重置、on_state_change 状态变更回调、滑动窗口失败追踪。

### 执行内容
- `async_call()` — 异步版 `call()`，接受 async callable，结构与 `call()` 对称
- `reset()` — 强制重置状态到 CLOSED，清空失败时间戳和 `_last_failure_time`
- `on_state_change` 回调 — `__init__` 新增 `Optional[Callable[[CircuitState, CircuitState], None]]` 参数，通过 `_set_state()` 统一触发，异常被 swallow
- 滑动窗口 — `_failures` 从 `int` 计数器改为 `collections.deque[float]` 时间戳队列，`_purge_old_failures()` 按 `window_duration`（默认 120s）清理过期记录，`_failures` 改为 property 返回窗口内失败数

### 技术要点
- `_set_state()` 内部方法统一处理状态转换 + 回调触发 + 相同状态去重
- 滑动窗口使用 deque 保证 O(1) 左端弹出，`_purge_old_failures()` 在 `_record_failure` 和 `_failures` property 中调用
- `_failures` 改为 property 保持向后兼容（现有代码通过 `cb._failures` 读取计数仍可工作）
- `_record_success()` 中 `self._failures = 0` 改为 `self._failure_timestamps.clear()`
- 所有新方法均在 `_lock` 保护下操作，保持线程安全
