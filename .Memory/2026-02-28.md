# 2026-02-28 项目记录

## Spec/Loop 模式三项优化：审查解析 + 截断修复 + 卡片布局

### 任务描述
修复 Spec 和 Loop 模式的三个用户体验问题：
1. 多视角审查输出频繁解析失败（所有视角显示"审查输出解析失败"）
2. 阶段输出被过度截断（format_phase_done 仅 500 字符）
3. 卡片布局信息缺乏层次感（进度/时长/标准/操作混在一个 markdown 块中）

### 执行内容

#### 1. 审查输出解析增强 — Level 2.5 "loose parsing"
- **新增 `parse_review_output_loose()`** (`src/utils/spec_utils.py`)
  - Strategy 1: JSON 数组格式解析（agent 直接输出 JSON）
  - Strategy 2: 逐行关键词对扫描（perspective keyword + PASS/FAIL 在同行或相邻行）
  - Strategy 3: Markdown 表格格式（`| 架构师 | PASS |`）
  - 支持中英文关键词、多种 verdict 表述（PASS/FAIL/通过/不通过/失败）
- **插入解析管线** (`spec_engine/engine.py`, `loop_engine/engine.py`)
  - 在 strict/tolerant 和 LLM 之间插入 loose parsing 作为 Level 2.5
  - `_parse_review_output`: strict → tolerant → **loose** → LLM → final fallback
- **收集 THOUGHT_CHUNK** (`_conduct_review` in both engines)
  - `on_review_event` 同时收集 TEXT_CHUNK 和 THOUGHT_CHUNK
  - 合并后传给解析器（部分 agent 审查结论在 thinking 中输出）
- **强化 `_build_review_prompt`** (both engines)
  - 使用 `<output_format>` XML tag 包裹格式要求
  - 增加 `<example>` few-shot 示例
  - 明确禁止使用 markdown 表格/JSON/编号列表

#### 2. 移除过度截断
- **`SpecReporter.format_phase_done`**: 移除 500 字符截断，完整输出
- **`StreamingCardManager._max_card_chars`**: 12222 → 28000（飞书卡片 JSON 上限约 30KB）
- **`close_streaming()`**: final card `max_chars=0`（不截断）

#### 3. 卡片布局优化
- **`build_deep_card` 新增可选参数**:
  - `status_line`: 状态行（notation-size，如 "🔄 循环执行中 · 循环 3 · 标准 1/5"）
  - `duration_line`: 时长行（与 status_line 合并为一个 notation-size 元素）
  - `criteria_section`: 验收标准独立区块（hr 分隔）
  - `footer_note`: 底部注释（notation-size）
- **卡片 elements 顺序**:
  ```
  📁 项目路径
  ─────────
  📊 进度条
  状态行 · 时长行           (notation-size)
  ─────────
  [主体内容]
  ─────────
  📋 验收标准列表           (独立 element)
  底部注释                  (notation-size)
  ─────────
  [按钮区]
  ```
- **Reporter 新增方法** (SpecReporter + LoopReporter):
  - `format_status_line(project)` — 一行状态摘要
  - `format_duration_line(project)` — 时长行
  - `format_criteria_section(project)` — 独立标准区块
- **Handler 回调接线** (`handlers/spec.py`, `handlers/loop.py`):
  - `on_cycle_start`, `on_phase_done`, `on_review_done` — 传递 status_line, duration_line, criteria_section
  - `on_project_done` — 传递 duration_line
  - `on_iteration_start`, `on_iteration_done`, `on_review_done` (loop) — 同步传递

### 技术要点
- Loose parsing 三策略优先级：JSON → 关键词对 → 表格
- JSON 策略支持多种 key 名：perspective/role/视角/角色 + verdict/result/结果/passed
- 关键词匹配支持 adjacent-line（verdict 不在同一行时向下看 2 行）
- THOUGHT_CHUNK 合并策略：text + "\n" + thought，避免正则跨 chunk 失效
- build_deep_card 参数全部可选，无传入时行为不变（完全向后兼容）
- StreamingCardManager: 流式过程中仍截断(28K)，但 final card 不截断
