# 2026-04-02 项目记录

## Thread 并发编程 R6-R10 修复与收敛

### 任务描述
延续 R6 审查发现的 4 个 MED 级问题，经 R7-R10 共 5 轮迭代审查修复，最终达成连续两轮无实质建议的收敛标准。

### 执行内容

#### R6 修复（4项）
1. **exit_mode 双重清理** — `get_thread_manager().remove(thread_id)` 从 `end_session` 之前移到 `try/finally` 块中的 `end_session` 之后，避免 `on_evict` 回调导致的双重 `end_session` 调用
2. **StreamingCard thread_root_id** — `StreamingCard` dataclass 新增 `thread_root_id` 字段，`create_streaming_card` 接受该参数，所有 `_build_buttons` 调用从 `card.thread_root_id` 读取而非 `threading.local`，解决 `ThreadPoolExecutor` worker 中按钮缺失 `thread_root_id` 的问题
3. **enter_mode 孤儿 session 清理** — 两个 `reply_message_with_id` 分支中，当 `response_id` 为 `None` 且 `_thread_enabled` 时，主动 `end_session` 清理孤儿 session 并向用户发送"话题创建失败，请重试"警告
4. **rebind_thread 冲突检查** — `rebind_thread` 在移动 session 前检查 `new_key` 是否已有 session，如有则先 `_end_session_unlocked` 再替换

#### R7 修复（3项）
1. **enter_mode 话题创建失败反馈** — 在 `_thread_enabled` 的 `elif` 分支添加 `reply_message` 错误提示，用户不再看到静默失败
2. **on_evict 测试覆盖** — 新增 `TestOnEvictCallback` 类 7 个测试（remove/remove_by_chat/evict_expired/close 触发回调、异常不传播、None 回调安全）
3. **rebind_thread 目标已存在测试** — 验证 rebind 时清理已有 session 的行为

#### R8 修复（1项）
1. **enter_mode _set_mode_on_project 条件对齐** — 新建分支（非 resume）条件从 `if not thread_id:` 改为 `if not thread_id and not _thread_enabled:`，与 resume 分支保持一致，避免 `_thread_enabled=True` 时污染 project mode state

### 技术要点
- **exit_mode 清理顺序**：用户反馈优先于资源清理（end_session → reply → remove → on_evict）
- **StreamingCard 显式传递 vs threading.local**：ThreadPoolExecutor worker 线程不共享 threading.local，必须在 dataclass 中显式存储 thread_root_id
- **_set_mode_on_project 一致性**：_thread_enabled 场景下新建和恢复分支必须保持相同的模式保护条件
- **mloop 收敛过程**：R6-R8 有实质建议（clean_rounds=0），R9-R10 连续无实质建议（clean_rounds=2/2）
