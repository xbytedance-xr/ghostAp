## Durable Ingress Phase 3 Task 6：锚定员工执行与真实 Slock Gateway

### 执行、恢复与锁边界

- 新增 `EmployeeDispatchCoordinator` 与 frozen dispatch/permit/environment contracts。Router dispatch、
  attempt binding、dispatch committed 在同一 Journal frame 锚定；公开 dequeue 只读，真实 dispatch 只由
  coordinator 驱动。0 个或多个 Slock、projection/credential/generation/membership/ACL 漂移均 durable
  fail-close，完整 replay 在提交锁之外，提交区只做 captured head/CAS。
- accepted attempt 最多一次调用既有 `_run_acp_session`。completed/failed/canceled/timeout/action_required
  原子写入数据面；unknown dispatch restart 后转 ACTION_REQUIRED 而不重跑。terminal/finalize CAS 冲突有界
  重试并复用 staged Blob，started attempt 必有唯一 terminal 或 action-required disposition。
- timeout/cancel 通过低依赖 ContextVar 贯穿 Sync ACP/Slock，保持 `_run_acp_session` 内部语义不变；
  provider 异常对外只暴露稳定类型，异常 context、日志、Journal 与返回文本不泄露 provider secret。

### Context、环境与权限闭包

- permit 冻结 tenant/agent/generation/BotPrincipal credential ref、persona、tool/model/profile/effort、
  permissions/capabilities、Context snapshot/watermark 与 renderer contract。Context 使用 canonical JSON
  区分 trusted instruction 和 untrusted thread/group/L1/L2，envelope 纳入 token reserve，`tokens_per_char`
  同时进入 snapshot hash，防止换算配置漂移后复用旧快照。
- 真实链路覆盖 Slock → factory → SyncACPSession → ACPSession.start → spawn boundary，并启动真实子进程
  验证 Manager/Vault/peer credential 环境变量不继承。员工 Git 与 Shell 均视为 write/shell-equivalent，
  permissions/capabilities 必须同时具备 `shell + git + file_write`；CR/LF、wrapper、危险 Git 输出/
  external-process 参数、非 canonical shape 与 safety exception 全部 fail-close。
- ACP `request_permission` 会把 canonical Git execute 分类到 Git policy 并补齐受控 cwd；真实
  `request_permission → SlockEngine filter` 回归证明能力可达，newline 注入不能借 Git policy 绕过。
