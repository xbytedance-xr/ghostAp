# 2026-06-07

## Workflow 契约统一：payload + template + injection + sanitization

**变更范围**：`src/card/events/payloads.py`, `src/card/ui_text.py`, `src/workflow_engine/constants.py`, `src/workflow_engine/templates.py`, `src/workflow_engine/script_gen.py`, `src/feishu/handlers/workflow.py`, `src/workflow_engine/runtime/runtime.js`

**背景**：Workflow 存在多维度契约漂移问题：(1) WorkflowRefItem TypedDict 字段与 `_WORKFLOW_BUTTON_FIELDS` 不统一；(2) 子 workflow refs 注入时没有 args/failure_policy 支持；(3) user_id 未在模板路径中隔离；(4) vm context 安全漏洞；(5) 错误消息泄漏 traceback 细节；(6) entry/help 文案缺少 Node.js >= 20 要求。

**实现**：
1. **payloads.py**：WorkflowRefItem 新增 `description/args/failure_policy` NotRequired 字段，保留 `path/hash` 为 NotRequired 以兼容老卡片；WorkflowConfirmCardValue 与 `_WORKFLOW_BUTTON_FIELDS` 对齐到同一字段集合，确保 `filter_workflow_button_value` 不会漏掉合法字段。
2. **constants.py**：`_discover_builtin_templates()` 动态扫描 `builtin_templates/*.js`；fallback 包括 adversarial-review/batch-migration/code-audit/refactor-pipeline/test-generation/doc-generation/performance-analysis。
3. **templates.py**：`resolve_safe_template_path(user_id=...)` 增加 user_id sanitization（拒绝 `/\\:..\\x00`），用户模板写入 USER_WORKFLOW_DIR，与 project/global 互斥。
4. **script_gen.py**：`_DANGEROUS_PATTERNS` 覆盖 `process[...]` bracket 访问、`.constructor.constructor(` 转义、`Function(` 构造器调用。
5. **workflow.py (handler)**：`_inject_workflow_refs_into_script()` 新增 args/failure_policy 注入；`_find_function_close_brace()` 用字符串/注释感知 brace 匹配修复 `main(args = {})` 默认参数的大括号误匹配；注入后调用 `validate_generated_script()` 重校验；role 按钮 `mobile_force_vertical=True`；`/wf_help` 与 `/wf_save` 错误路径使用统一 `_node_version_required_text()` 和 `_strip_internal_details()`。
6. **runtime.js**：`JsonRpcError` 类保留 `code/message/data`；`sandboxWrapHostFn()` 设置 `Object.setPrototypeOf(fn, null)` 并 `Object.freeze()`；sandboxGlobals 冻结 Object/Function/Array/String/Number/Boolean/Error/Promise 原型链。
