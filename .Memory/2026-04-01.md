## 基于 Claude Code Agent Loop 分析的 Loop/Spec 引擎优化

### 任务描述
读取飞书文档《Claude Code Agent Loop 深度解析》，借鉴其架构模式（防漂移 TodoWrite、wU2 上下文压缩、Session 管理）对 GhostAP 的 LoopEngine 和 SpecEngine 进行优化。

### 执行内容

#### R1-1: LoopEngine 接入 LoopContextManager + 防漂移上下文锚点注入
- **问题**: `LoopContextManager` 已在 `models.py` 中实现（三级压缩：远程/中程/近期），但从未被 `engine.py` 引用
- **修复**: 
  - 修复 `LoopContextManager.build_context_prompt()` 中 `record.role=None` 导致的 AttributeError（2处）
  - 在 `engine.py` 中初始化 `LoopContextManager`，每轮迭代后记录，构建 prompt 时注入三级压缩历史
  - 新增**防漂移目标锚点**：每轮 prompt 开头注入 `## 原始目标\n{requirement.goal}` 防止多轮后 LLM 遗忘目标
- **文件**: `src/loop_engine/models.py`, `src/loop_engine/engine.py`

#### R1-2: LoopEngine 增强收敛检测
- **问题**: 原收敛检测仅检查最近 N 轮输出长度 < 50 字符，过于简单
- **增强**: 新增第二维度 — 如果 `criteria_progress` 的 `satisfied_count` 在窗口内未变化 AND 所有迭代均失败 → 判定收敛
- **文件**: `src/loop_engine/engine.py`

#### R1-3: SpecEngine 阶段间传递结构化产物
- **问题**: 各阶段（Spec→Plan→Task→Build）间传递完整 LLM 原始输出文本，造成严重上下文膨胀
- **优化**: 当 artifact 解析成功时（如 `SpecArtifact.goals` 非空），使用 `json.dumps(artifact.to_dict())` 替代原始文本
- **向后兼容**: artifact 为空时自动回退到原始文本
- **文件**: `src/spec_engine/prompts.py`（3个函数签名扩展）, `src/spec_engine/engine.py`（3处调用修改）

#### R1-4: SpecEngine 循环间 Session 重建
- **问题**: 所有 phase 共享一个 ACP Session 跨所有 cycle，10 cycle 后积累 60+ 对话轮次
- **优化**: 新增 `spec_rebuild_session_between_cycles` 配置（默认 True），非终止 cycle 结束后调用 `_recreate_session_best_effort()` 重建 Session
- **副作用修复**: 修复 4 处测试 mock settings 缺少新属性导致 MagicMock 返回 truthy 触发真实 ACP Session 创建；5000 cycle 压力测试从 120s 超时恢复到 9s
- **文件**: `src/config.py`, `src/spec_engine/engine.py`, `src/spec_engine/session_utils.py`（日志措辞优化）, `tests/test_spec_engine.py`（4处 mock 修复）

### 技术要点
- **Claude Code 防漂移模式**: TodoWrite 每次工具调用后注入当前 TODO 状态，GhostAP 对应实现为 `goal_anchor` 注入原始目标到每轮 prompt
- **三级上下文压缩**: 远程（单行摘要）、中程（简要总结）、近期（完整输出），按 token 预算动态分配
- **MagicMock 陷阱**: 用 MagicMock 做 settings 时，未定义的属性返回 truthy MagicMock 对象，可能意外触发条件分支
- **Session 重建位置**: 放在 for-loop 的终止检查之后（break 之后），确保只对继续执行的 cycle 触发
