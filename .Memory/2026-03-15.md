## SpecEngine 循环策略可配置化（spec_min_cycles）

### 任务描述
SpecEngine 的 `ContinuationPolicy.min_cycles` 原为硬编码 2，且在单测使用 `MagicMock` settings 时可能被错误解析为 1（`int(MagicMock()) == 1`），导致在 `spec_convergence_window=1` 时提前触发收敛终止。

### 执行内容
1. **新增配置项**：在 `src/config.py` 增加 `spec_min_cycles`（默认 2），用于控制最少循环次数。
2. **修复 SpecEngine 硬编码**：在 `src/spec_engine/engine.py` 使用 `spec_min_cycles` 来构建 `ContinuationPolicy`。
3. **修复 settings int 读取对 MagicMock 的兼容性**：收紧 `_get_int_setting()` 的类型处理，仅接受 int/float/str，避免 `MagicMock` 被 `int()` 误转换成 1。
4. **修复收敛窗口边界**：`spec_convergence_window < 2` 时禁用收敛判断，避免“窗口=1”导致永远视为无进展而提前终止。
5. **验证**：回归运行 Deep/Loop/Spec 共 292 个测试用例全部通过。

### 技术要点
- **配置与策略解耦**：将“至少跑几轮”从代码硬编码下沉到配置项，既保留默认行为（2 轮用于 discovery），也允许单轮任务/测试场景。
- **Mock 兼容性**：`MagicMock` 具备 `__int__`，不做类型防护会引入隐式值漂移，需在读取配置处显式规避。
