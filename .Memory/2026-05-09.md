## 2026-05-09 11:40:00 修复 Worktree 卡片按钮交互未触发

### 任务描述

### 执行内容
1. **根因定位**
   - 对照当前 Worktree HTML 预览和真实渲染路径，确认工具选择卡由 `src/card/render/worktree.py` 直接生成内嵌按钮。
   - 这些内嵌按钮仅携带 `value`，缺少飞书 Schema 2.0 callback 所需的 `behaviors: [{"type": "callback", "value": ...}]`。
   - 后续 Worktree 阶段的 `ButtonSpec` 通用渲染路径同样只输出 `value`，存在下一阶段按钮继续不可交互的风险。

2. **修复内容**
   - `src/card/render/worktree.py` 新增 `_callback_button()`，让工具添加、模型选择、移除、清空、确认按钮都输出 `value` + `behaviors`。
   - `src/card/render/buttons.py` 的通用 callback 按钮输出 `behaviors`，覆盖 Worktree 后续确认/执行/合并/清理按钮。
   - 保留原 `value` 字段，避免破坏现有路由和测试中对 legacy value 的读取。

3. **测试覆盖**
   - `tests/test_worktree_render_direct.py` 增加断言：Worktree 工具选择、模型选择、移除、清空、确认按钮必须带 callback behaviors，且与 `value` 一致。
   - `tests/test_card_render_components.py` 增加通用按钮 callback behaviors 断言。

### 技术要点
- Schema 2.0 卡片按钮需要 `behaviors` 才能作为前端 callback 交互生效；仅保留 `value` 不足以保证客户端触发回调。
- 兼容策略是同时输出 `value` 和 `behaviors`，让旧的服务端解析与新的客户端交互字段同时存在。
