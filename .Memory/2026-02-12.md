# 2026-02-12 项目记录

## Shell 命令执行无限递归修复

### 任务描述
用户反馈 Shell 命令执行后卡住无返回。

### 执行内容

**根因分析**：`SystemHandler.submit_shell_command` 的 `_run` 回调通过 `self.ctx.message_callback` 调用 `Application.handle_message`，后者又调用 `_submit_shell_command`，形成无限递归任务创建循环。Shell 命令（SandboxExecutor.execute）从未被实际调用。

调用链：
```
submit_shell_command._run()
  → message_callback (= Application.handle_message)
    → _submit_shell_command
      → 新任务._run() → message_callback → ... (无限循环)
```

**修复方案**：新增 `SystemHandler.execute_shell_and_reply()` 方法，直接调用 `SandboxExecutor.execute()` 并回复结果，打破递归循环。

**修改文件**（3 个）：
1. `src/feishu/handlers/system.py` — 新增 `execute_shell_and_reply()`；`submit_shell_command._run` 改为调用该方法
2. `src/feishu/ws_client.py` — `shell_fast_tracked` 路径和多任务步骤路径改为调用 `execute_shell_and_reply`
3. `src/main.py` — `Application.handle_message` 改为调用 `execute_shell_and_reply`

### 技术要点
- 无限递归的根因：`message_callback` 是 legacy 回调，在 Handler 架构拆分后变成了中间人，但 `submit_shell_command` 仍然通过它间接回调自己
- 修复策略：让所有 shell 执行路径直接调用 `SandboxExecutor.execute()` + `reply_message()`，而非通过 `message_callback` 间接调用
- 792 测试全通过
