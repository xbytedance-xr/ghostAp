## Deep 卡片副标题与折叠阈值优化

- 问题：Deep 多任务卡片的任务名会同时出现在 header title 与 subtitle，移动端看起来像“小字重复标题”；任务列表在 `total > 12` 时提前折叠已完成任务，工具调用摘要也只展示 6 条详情后折叠。
- 修复：Deep v2 header 的 subtitle 改为展示整体运行总耗时，格式为 `H时MM分SS秒`，耗时来源继续使用 CardSession 注入 `_now` 后推导的 `RuntimeStats.elapsed_seconds`，不新增定频刷新。
- 修复：任务列表折叠阈值与工具调用摘要阈值分离；任务三桶各自尽量展示到 50 条后才折叠，工具调用摘要详情提升到 12 条后折叠。

## Spec 卡片 JSON 可读性与阶段状态流转修复

- 问题：Spec 模式卡片在 `SPEC/PLAN/TASK` 等结构化阶段会把模型输出的完整 JSON artifact 作为普通文本实时流出；阶段完成时还会把 raw `output` 写入 `PhaseBlock.content`，最终在 Feishu phase panel 中显示大段 JSON。
- 问题：同一 cycle 每个 phase 都追加独立 `phase_{cycle}_{phase}` 状态块，旧的“✅规格定义 → ⬜方案规划 → ...”快照会留在卡片顶部，看起来状态条没有推进。
- 修复：`SpecStreamProcessor` 对 `SPEC/PLAN/TASK` 的 `TEXT_CHUNK` 不再转发到卡片正文，raw 输出仍留在 engine tracker/artifact/persistence 路径；`PHASE_DONE` 只向 phase reducer 传 `SpecReporter.format_phase_done_content()` 生成的可读摘要。
- 修复：review 完成时 phase panel 展示“多视角审查完成”状态摘要，详细审查结果改走普通文本块；phase 事件新增 `content/subtitle` payload 以同步顶部状态和 header subtitle。
- 修复：phase reducer 在同一 cycle 内只保留一个当前 phase panel，新 phase start 会替换上一阶段状态块，避免旧状态快照固定在卡片顶部；同时补齐 Spec phase 的 footer 文案映射。
