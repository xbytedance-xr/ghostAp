## TTADK CLI 模式 prompt 传递修复

### 任务描述
TTADK 模式完全不能用——`SyncTTADKCLISession.send_prompt()` 将用户 prompt 作为 `ttadk code` 的位置参数，但 `ttadk code` 不接受位置参数（报错 "too many arguments for 'code'"）。

### 执行内容

**根因分析**
- `cmd.append(text)` 把 prompt 追加到 `["ttadk", "code", "-t", tool]` 末尾
- `ttadk code` 的 CLI 接口只接受选项（`-t`, `-m`, `-a`），不接受位置参数
- 正确方式是通过 `-a` (args passthrough) 将参数传递给底层工具

**工具调研**
- coco/claude: `-p` (print mode) 进入非交互模式，prompt 作为位置参数
- gemini: `-p/--prompt "text"` 进入 headless 模式
- codex: prompt 直接作为位置参数，无 `-p` print mode
- ttadk 的 `-a` 参数做 shell-like 解析（尊重引号），因此 `shlex.join()` 可正确编码

**修改文件**
1. `src/agent_session.py`:
   - 添加 `shlex` import
   - 新增 `_TTADK_PRINT_MODE_TOOLS = frozenset({"coco", "claude", "gemini"})`
   - 新增 `_build_ttadk_passthrough_prompt()` 函数：print-mode 工具用 `-p prompt`，其他用纯位置参数
   - `send_prompt()` 中 `cmd.append(text)` → `cmd.extend(["-a", _build_ttadk_passthrough_prompt(...)])`
   - 扩展 `_TTADK_PREAMBLE_PATTERNS` 新增 3 个模式（⚠/❯ emoji、Continuing/Login/Launching 行）
   - 添加 `Popen` 前 debug 日志 `logger.debug("[TTADK:CLI] cmd=%s cwd=%s", ...)`
2. `tests/test_ttadk.py`:
   - `TestBuildTTADKPassthroughPrompt` (8 tests): 验证各工具的参数格式化
   - `TestSyncTTADKCLISessionCmdArgs` (3 tests): 验证 send_prompt 命令构建正确性
   - `TestTTADKPreambleNewPatterns` (4 tests): 验证新增 preamble 模式

### 技术要点
- `ttadk code -a "..."` 做 shell-like 解析，`shlex.join()` 生成正确的引号格式
- coco/claude/gemini 的 `-p` flag 让工具在无 TTY 环境下也能执行（输出结果后退出）
- codex 没有 `-p` print mode，但 subprocess 环境下模型选择需要 `-m` 参数避免交互卡住
- 多个 `-a` 标志不被 ttadk 支持（只有最后一个生效），必须用单个 `-a` + 完整参数串

## TTADK 卡片 banner 过滤 + 标题增强

### 任务描述
两个问题：(1) TTADK ASCII art banner 第 3 行含单引号 `'`，未被 preamble 正则过滤，残留在飞书卡片中；(2) 卡片标题只显示 `🎮 ghostAp · TTADK`，不显示当前代理的工具名（如 claude/gemini）和模型名。

### 执行内容

**1. Preamble 过滤修复**

**2. 卡片标题增强**

- `src/feishu/handlers/programming.py`: 基类新增 `_get_ttadk_tool_display()` / `_get_ttadk_model_display()`；TTADKModeHandler 覆写前者复用 `_get_agent_type_override` 去前缀
- `src/card/builders/core.py`: `_build_header_title()` 从 `project.ttadk_tool_name/model_name` 构建非流式卡片标题
- 标题格式示例：`🎮 ghostAp · TTADK · claude(glm-5)`
