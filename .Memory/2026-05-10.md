## 飞书编程卡片 v2 重设计首批实现
### 任务描述
基于已确认的 brainstorm、双屏 HTML mockup、`docs/superpowers/specs/2026-05-10-card-redesign-design.md` 和 `docs/superpowers/plans/2026-05-10-card-redesign-v2.md`，开始实现飞书编程卡片 v2。首批范围聚焦低耦合基础能力：metadata foundation、header、task list、tool panel 展开规则、footer hint。

### 执行内容
- `CardMetadata` 增加 `card_sequence`、`session_started_at`、`working_dir`、`is_subagent`、`parent_card_seq`、`final_state_for_freeze`、`frozen`、`frozen_total_elapsed` 等 v2 字段；`CardSession` 暴露对应只读属性；`CardEvent.card_split()` 增加可选 `bridge_phrase`。
- `src/card/render/header.py` 支持 v2 两行编程 header：第一行展示项目、工具、卡片序号、模型；第二行展示目录/父卡来源与运行时长；冻结卡展示 `已封存`、灰色 template 和 final elapsed。
- `src/card/render/task_list.py` 将 task list 改成三段常开：进行中 / 已完成 / 未处理；compact sticky 路径不再只显示当前任务；大任务量下折叠已完成尾部、未处理只显示前 5 个。
- `src/card/render/tools.py` 在渲染层强制只有 `status=active` 且 `is_latest_active=True` 的工具展开，避免 stale latest flag 导致已完成/失败工具展开。
- `src/card/render/footer.py` 增加 `render_now_tool_hint()`，footer 展示当前运行工具一句话 hint；subagent metadata 打开时展示 model/tool/parent badge。

### 技术要点
- 计划文档里 `CardSession` 数据模型路径与当前实现不一致：真实渲染持久契约在 `CardMetadata`/`CardState`，所以 v2 字段放入 `CardMetadata`，并通过 `CardSession` 属性透出，避免改动 session 构造链路。
- v2 header 仅在 metadata 带编程上下文时启用，保留普通静态卡片的 legacy header 行为，降低测试卡和非编程卡误伤。
- task list v2 与旧 compact 行为冲突，相关旧测试已改为断言三段常开，这是本次设计确认后的语义变化。
- 完整回归中 `tests/test_validate_mode.py::TestValidateNoTombstoneTrigger::test_validate_does_not_trigger_tombstone` 曾出现一次 subprocess 30s 偶发超时，单独复跑通过；最终阶段全量通过。
