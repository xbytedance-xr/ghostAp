# 2026-07-10

## WF 报告展示压缩与启动失败熔断

### 背景
- `/wf` 终态卡片需要继续保持可读的紧凑摘要，同时 HTML 报告需要承担完整执行过程、阶段与 agent 明细展示。
- 当某个 ACP 工具启动失败时，Workflow 后续 agent 调用仍可能反复尝试同一不可用工具，每次都消耗启动超时预算。

### 改动
- `src/workflow_engine/reporting.py`：
  - HTML 报告新增顶部统计、状态徽标、阶段时间线和 agent 明细列表。
  - 报告章节改为更清晰的 section body 和 sticky 搜索工具栏，仍保持自包含 HTML 并对动态内容做 `html.escape()`。
- `src/workflow_engine/renderer.py`：
  - 完成卡片统计压缩为 4 列单行摘要。
  - 执行过程摘要前置，完整报告存在时卡片仍保留截断后的结果摘要，并把附件发送状态放在后面。
- `src/workflow_engine/executor.py` 与 `src/workflow_engine/errors.py`：
  - 检测 `ACPStartupError` 异常链，按工具在单次 Workflow run 内建立启动失败熔断。
  - 同一工具再次被调用时直接返回 circuit breaker 错误，避免重复等待启动超时。
  - 将 circuit breaker / ACP startup failure 归入永久错误关键词。
- `src/workflow_engine/script_gen.py`：
  - fallback 生成脚本的 verifier 固定使用主编排工具，减少评审工具误落到候选中不可用工具的概率。
- 更新 `tests/test_workflow_renderer.py` 对新卡片文案和报告摘要策略的断言。

- `git diff --check` → 通过。

### 后续风险
- 启动失败熔断作用域是单次 Workflow run，适合避免本轮重复超时；如果工具后续恢复，需要启动新的 Workflow 才会重新尝试。
- HTML 报告视觉结构更丰富，仍需要真实飞书附件链路里的移动端阅读体验反馈继续校准。

## WF 结果简报卡去截断

### 背景与根因
- 终态卡同时承担结果正文和状态摘要，`renderer.py` 对 `project.result` 使用 4,000/2,000 字符上限，再由 `_enforce_card_size()` 做第二次按字符裁剪或尾部元素删除。
- 结果是同一卡片多处出现“内容已截断”，用户无法确认丢失的是结论、证据还是风险；7 月 9 日已有的完整 HTML/Markdown 报告只能保证归档，不能替代卡片上的关键行动信息。
- 两套 `on_done()` 异常降级仍回复 `result[:500]`，卡片投递失败时会再次把不完整原始正文发到话题。

### 改动
- 新增 `src/workflow_engine/result_brief.py`：
  - Pydantic 类型化 `WorkflowResultBrief`、verdict/severity 和完整 `BriefItem`。
  - 优先验证顶层 `card_summary`；旧结果只映射 `summary/conclusion/verification/findings/issues/risks/deliverables/artifacts/next_steps/recommendations` 等已知字段，不递归猜测未知对象。
  - 纯文本或无法可靠识别的结果使用“任务已完成，完整结果见报告”中性结论。
  - 按 UTF-8 预算选择完整条目；过长或超量条目整体省略并累计“另有 N 条”，绝不切半句。
- `src/workflow_engine/renderer.py`：
  - 完成卡只消费标准化简报，信息顺序改为结论→关键发现→验证→交付物→下一步，空栏目隐藏。
  - 统计改为耗时/阶段/验证/高风险或交付物；`needs_attention` 使用“需修正”终态。
  - 执行过程移到结果后的折叠面板，报告状态置底。
  - 结果简报在 Markdown 转义前执行 agent-output 防泄漏检查。
  - 完成卡超过 28KB 时重建最小卡，只保留完整结论、遗漏数量和报告/错误状态，不经过字符截断。
- `src/workflow_engine/script_gen.py`：
  - AI 脚本生成提示要求成功结果保留完整业务产物并返回 `card_summary`。
  - 静态 fail-closed 脚本和 `generate_simple_script()` 通过 `completionEnvelope()` 输出同一契约；执行错误仍保留原有顶层失败语义。
- `WorkflowScriptMixin` 增加共享安全终态降级，两套 callback 不再回复 `result[:500]`；只说明结果卡失败和完整报告状态。
- 测试基础设施最小修复：action registry 路径从当前 checkout 推导；并发压力测试仍执行 80 次写入，但用 8 个 writer 线程避免受限机器把线程创建本身当成被测行为。
- 设计与预览：`docs/superpowers/specs/2026-07-10-workflow-result-brief-design.md`、`ux/workflow-result-brief.html`。
