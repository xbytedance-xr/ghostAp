## 日志 WARNING 修复与优化

### 任务描述
检查 logs.log 中的 WARNING/ERROR 日志，修复无效行为并优化日志级别。

### 执行内容
分析出 4 类 WARNING，其中 ttadk update→upgrade 已在前序任务修复，本次处理剩余 3 类：

1. **"No running loop for throttling" (25次)** — `src/feishu/handlers/base.py:192`
   - 原因：`patch_message(throttle=True)` 在 sync worker thread 中调用 `asyncio.get_running_loop()` 失败，回退到立即 patch
   - 修复：`logger.warning` → `logger.debug`，这是预期的 sync 回退行为，不应污染 WARNING 日志

2. **"ProbeStrategy timed out" (1次)** — `src/ttadk/strategies.py:302`
   - 原因：首次 probe 尝试(2.5s超时)经常超时，由上层 fetcher 重试后成功
   - 修复：`logger.warning` → `logger.info`，超时是正常路径的一部分

3. **"添加表情失败: reaction type is invalid" (3次)** — `src/feishu/emoji.py`
   - 原因：多个 emoji_type 值不在飞书官方表情列表中
   - 修复（对照官方文档 https://s.apifox.cn/apidoc/docs-site/532425/doc-398404）：
     - `ROCKET = "Rocket"` → `"Fire"` （官方列表无 Rocket/ROCKET）
     - `CROSS_MARK = "Skull"` / `ERROR = "Skull"` → `"SKULL"` （官方为全大写）
     - `ONE_SEC = "OneSec"` / `BUSY = "OneSec"` → `"OneSecond"` （官方为 OneSecond）

### 技术要点
- 飞书 emoji_type 大小写敏感，且不支持自定义类型
- sync worker thread 中无 asyncio event loop 是正常情况，不应作为 WARNING
- ProbeStrategy 超时是 fallback 链的正常环节

## TTADK 模型选择跳过与双重鉴权修复

### 任务描述
用户反馈两个问题：
1. 选择 TTADK 工具后直接进入编程模式，跳过了模型选择步骤
2. TTADK 触发了两次 OAuth 鉴权（用户已授权，不应重复鉴权）

### 执行内容

**Fix 1: 非 YOLO 模式下不跳过模型选择卡片** — `system.py:L872-880`
- 原因：`handle_select_ttadk_tool` 中 `_pick_ttadk_auto_model` 无论 YOLO 开关状态都会尝试自动选择模型（如项目已保存模型名或有 is_default 的模型），导致非 YOLO 模式也跳过了模型选择卡片
- 修复：将 auto_model 选择逻辑整体放入 `if yolo_enabled:` 条件块内

**Fix 2: auto_update_ttadk 改为后台异步执行** — `manager.py:L1684-1717`
- 原因：`auto_update_ttadk()` 同步执行 `subprocess.run(["ttadk", "upgrade"])`，阻塞 `/ttadk` 命令主流程
- 修复：改为 `threading.Thread(daemon=True, name="ttadk-auto-upgrade")` fire-and-forget 模式
- 附加：`threading.Lock` 保护 `_ttadk_update_attempted` 竞态条件

**Fix 3: sandbox 鉴权目录保留** — `env_sandbox.py`
- 根因定位：`build_ttadk_subprocess_env` 覆盖 HOME 为 sandbox 路径，但 ttadk 的 OAuth token 存储在 `$HOME/Library/Preferences/bytesso-nodejs/device-shared.json`，sandbox 覆盖后子进程找不到已有鉴权 token → 触发 OAuth 弹窗
- 修复：新增 `_symlink_auth_dirs()` 函数，在 sandbox 创建时将真实 HOME 下的鉴权目录（`Library/Preferences/bytesso-nodejs`）符号链接到 sandbox 中
- 效果：sandbox 子进程共享真实 HOME 的鉴权信息，不再触发重复 OAuth

### 技术要点
- ttadk OAuth token 路径：`$HOME/Library/Preferences/bytesso-nodejs/device-shared.json`（macOS）
- sandbox 覆盖 HOME 会导致所有依赖 $HOME 路径的鉴权信息丢失
- 符号链接方案既保留 sandbox 写隔离目标，又共享鉴权 token
- auto_update 的返回值在调用方未使用，可安全改为 fire-and-forget
- `_AUTH_PRESERVE_DIRS` 使用 `sys.platform == "darwin"` 条件限定 macOS 路径
