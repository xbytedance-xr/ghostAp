# 2026-04-25 项目记录

## 卡片系统统一重构 Phase 1 + Phase 2

### 任务描述

将编程模式（Coco/Claude/Aiden/Codex/Gemini/TTADK）的流式卡片和引擎模式（Deep/Loop/Spec）的引擎卡片统一为同一套卡片模板。Phase 1 统一布局，Phase 2 引擎折叠面板。

### 执行内容

#### Phase 1: 统一卡片布局构建器

1. **新增 `CardLayoutSpec` 数据模型** (`src/card/models.py`)
   - 统一描述卡片布局的所有参数（通用字段 + 流式状态栏 + 引擎特有区域 + 内容 + 渲染选项）
   - 支持 `content_markdown`/`content_elements` 二选一（纯 markdown 或预构建折叠面板 elements）
   - 支持 `buttons`/`button_elements` 二选一（普通按钮列表或预构建 grouped layout）

2. **创建 `UnifiedCardLayout` 构建器** (`src/card/builders/layout.py`, 新文件)
   - 统一超集布局：路径 → 元数据 → 警告 → 分隔线 → 进度条 → 图片 → 主内容 → 验收标准 → 脚注 → 按钮
   - 自动适配流式模式（路径+状态栏合并）和引擎模式（路径独立+元数据行+分隔线）
   - 辅助函数 `_build_path_and_meta()` 和 `_build_engine_meta()` 处理两种路径

3. **迁移 `StreamingCardManager`** (`src/card/streaming.py`)
   - `_build_elements()` → 构造 `CardLayoutSpec` → `UnifiedCardLayout.build()`
   - `_build_structured_elements()` → 同上，通过 `content_elements` 传入预构建元素
   - 删除 ~80 行手动元素组装代码

4. **迁移 `DeepBuilder.build_engine_card()`** (`src/card/builders/deep.py`)
   - 保留 compact/expanded/full 内容截断逻辑、验收标准截断、按钮构建（grouped layout）
   - 将最终元素组装委托给 `UnifiedCardLayout.build()`
   - 删除 ~50 行手动元素组装代码

#### Phase 2: 引擎卡片折叠面板

5. **`EngineCardState.rendered_content` 字段** (`src/card/models.py`)
   - 新增 `rendered_content: Optional[object]` 字段
   - 当存在时，`DeepBuilder` 使用 `to_elements(collapsible=True)` 替代纯 markdown

6. **`DeepBuilder` 支持结构化内容** (`src/card/builders/deep.py`)
   - 非 compact 模式下，如果有 `rendered_content`，使用 collapsible panels
   - 异常时自动降级为纯 markdown

7. **`DeepRenderer` 产出结构化内容** (`src/feishu/renderers/deep_renderer.py`)
   - `on_event()` 改用 `process_event_structured()` 保存 `RenderedContent`
   - `_maybe_stream_update()` 和 `on_project_done()` 传入 `rendered_content`
   - 受 `engine_collapsible_enabled` 配置开关控制

8. **新增配置开关** (`src/config.py`)
   - `engine_collapsible_enabled: bool = True`

### 技术要点

- **增量迁移策略**：先抽取统一构建器，再逐个迁移调用方，每步都运行全量测试
- **向后兼容**：`UnifiedCardLayout` 的所有字段都有默认值，现有调用方不需要传全部字段
- **引擎按钮特殊处理**：引擎的 grouped layout（控制按钮并排）通过 `button_elements` 预构建传入，不走 `build_responsive_layout`
- **结构化内容降级**：`rendered_content.to_elements()` 异常时自动降级为纯 markdown

## 卡片优化 Phase 3: pokoclaw 对齐 + 内容消失 bug 修复

### 任务描述

将 ghostAp 的消息卡片视觉与交互体验对齐 pokoclaw 项目（CardKit Schema 2.0）。涉及截断逻辑统一、终态标记、footer 状态行、payload 预算、字符串集中化等 12 项功能点。同时修复一个关键 bug：引擎模式卡片只显示进度但无文本内容。

### 执行内容

#### Step-04: 截断逻辑统一到 truncation.py
1. **新建 `src/card/truncation.py`** — 统一截断模块
   - `truncate_card_string(value, max_chars, max_lines)` — 先按行截断再按字符截断
   - `truncate_bash_output(value, max_chars, max_lines)` — bash 专用截断
   - `cap_reasoning_tail(value, max_chars)` — 保留尾部 max_chars 字符
   - `truncate_terminal_message(value, max_chars)` — 终态消息截断
   - 所有阈值从 `TRUNCATION_LIMITS` 读取

2. **迁移 `system.py` Shell 截断** — `build_shell_result_card()` 的 stdout/stderr 保持原有简单切片逻辑（SHELL_STDOUT_MAX=16000, SHELL_STDERR_MAX=8000），不迁移到 truncation.py（阈值差异太大）

#### Step-16: 全量回归测试
- 2996 passed，1 pre-existing failure（`test_spec_settings_defaults` 期望 `spec_max_cycles=500`，实际 `2000`）

#### Step-17: 字符串集中化审查
1. **新增常量** `UI_TEXT["system_ttadk_btn_reenter"] = "🔄 重新进入TTADK"`
2. **修复 5 处硬编码中文字符串**：
   - `system.py:518` — `"⚠️ TTADK 暂不可用"` → `UI_TEXT["system_ttadk_unavailable_title"]`
   - `system.py:502` — `"🔄 重新进入TTADK"` → `UI_TEXT["system_ttadk_btn_reenter"]`
   - `system.py:534` — `"继续进入TTADK"` → `UI_TEXT["system_ttadk_btn_continue"]`
   - `worktree.py:422` — `"执行中"` → `UI_TEXT["system_status_executing"]`
   - `project.py:523` — `" 模式"` → `UI_TEXT["system_mode_label"]`
3. **layout.py** — `f"图片 {i+1}"` → `UI_TEXT["image_alt_text"].format(index=i+1)`，清理冗余 local imports

#### 关键 Bug 修复：引擎卡片内容消失
- **根因**：`deep.py:345-352` 中，`state.rendered_content.to_elements()` 返回空列表 `[]` 时，代码无条件将 `use_content_markdown = None`，丢弃了原本有效的文本内容
- **触发路径**：`RenderedContent.to_elements()` 在流式早期或特定事件组合下返回 `[]` → `content_elements = []`（非 None）→ markdown 被丢弃 → layout 中 `if spec.content_elements is not None:` 对 `[]` 判 True → extend 空列表 → 主内容区域空白
- **修复**（两层防御）：
  - `deep.py`：仅当 `to_elements()` 返回非空列表时才替代 markdown，否则保留 markdown fallback
  - `layout.py`：将 `is not None` 改为 truthiness 检查 `if spec.content_elements:`
