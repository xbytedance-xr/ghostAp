## Spec 完成度与方向把控角色

- 背景：Spec 模式多角色审查经常只从各专业视角给建议，缺少一个独立角色专门判断当前循环是否仍贴合用户原始方向、是否高估完成度、是否具备停止条件。
- 修改内容：
  - `src/spec_engine/review_roles.py`
    - 新增 `completion_control` 角色（显示名“完成度与方向把控”），作为 blocking review role，关注用户原始目标一致性、验收/构建/测试证据、完成度高估风险和下一轮方向。
    - 编程任务保留原 5 个固定软件审查角色，并额外加入 `completion_control`；动态角色只使用剩余 slot，低 `total_roles_max` 时优先保留固定角色 + 完成度角色。
    - 写作/研究/设计/通用任务也保留该角色，并在角色 cap 内为它预留 slot。
  - `src/spec_engine/adaptive_review.py`
    - 单角色 review prompt 增加 Spec/Plan/Task/Build 阶段输出，给完成度角色真实证据来判断是否偏离方向或完成度不足。
  - `src/spec_engine/review_strategy.py`
    - adaptive role planner 异常 fallback 时也使用固定 5 角色 + 完成度角色，避免规划失败时丢失完成度把控。
  - 测试覆盖角色规划、strategy 运行、fallback、完成度角色 blocking 建议、prompt 阶段证据注入。

## Workflow 2-Step Combined Card Refactor

### What Changed
Refactored the `/wf` (Workflow) mode from a 4-step orchestration flow
(agent→tools→roles→confirm) into a 2-step flow (combined selection→confirm)
to fix multiple blocking issues preventing workflow tasks from starting.

### Changes Made

1. **Fixed Feishu card schema errors** (`src/card/render/buttons.py`,
   `src/feishu/handlers/workflow.py`)
   - Removed `vertical_spacing` property from `column_set` elements in
     `_layout_buttons()` and `build_responsive_button_row()` — Feishu API
     returned error 200621 ("unknown property vertical_spacing on column_set").
   - Removed `vertical_spacing` from 3 `collapsible_panel` elements in
     workflow handler.

2. **Fixed engine.project None initialization** (`src/workflow_engine/engine.py`)
   - `WorkflowEngine.__init__` initialized `self._project = None`, but
     handler code checks `if engine.project:` before setting pending state.
     Changed to `self._project = WorkflowProject()`.

3. **Implemented stale state auto-reset** (`src/feishu/handlers/workflow.py`)
   - Previously, if user abandoned a card mid-flow, the `AWAITING_*` state
     persisted forever, blocking new workflows with "已有 Workflow 等待操作".
   - Now auto-resets stale pending states to IDLE in both
     `handle_show_workflow_menu` and `start_workflow`.

4. **Built 2-step combined selection card** (`src/feishu/handlers/workflow.py`,
   `src/card/ui_text.py`)
   - Replaced `_show_agent_selection_card` internals to call
     `_send_combined_selection_card` which merges agent + tool + role + budget
     selection into one card (step 1), followed by confirm/execute (step 2).
   - Updated `_build_workflow_stepper` from 4-step to 2-step.
   - `handle_workflow_select_agent` now re-renders combined card instead of
     navigating to separate tool selection step.
   - `handle_workflow_confirm_roles_and_generate` accepts `AWAITING_AGENT_SELECT`
     state for direct confirm from combined card.

5. **Updated tests** (`tests/test_workflow_orchestrator_select.py`,
   `tests/test_workflow_confirm.py`)
   - Updated 13 tests in orchestrator_select to match combined card flow.
   - Updated 8 integration/e2e tests in confirm flow to use 2-step path.
