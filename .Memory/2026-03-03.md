# Memory: 2026-03-03

## 关键修复与优化

### 1. 修复 Spec 模式任务恢复逻辑
- **问题**: `SpecHandler.recover_spec_task` 在从 `SpecTaskState` 恢复项目快照后，错误地调用了 `engine.execute()`，导致项目状态被重新初始化，之前的进度丢失。
- **修复**: 修改为调用 `engine.resume()`，并在恢复前将项目状态置为 `PAUSED` 以通过 `resume` 的前置检查。同时修正了 `SpecEngine.resume` 中计算 `start_cycle` 的逻辑，使其能够正确处理 compact state（仅保留最近 N 个 cycle）的情况，避免 cycle 编号回退或覆盖。

### 2. 修复 Spec 模式 WebSocket 消息处理异常
- **问题**: `FeishuWSClient` 在处理纯文本消息时，由于 `project_id` 变量作用域问题导致 `NameError`。
- **修复**: 优化了 `project_id` 的获取逻辑，优先从 `project` 对象获取，并增加了从 `task_ctx` 回退获取的机制。

### 3. 解决 Claude Nested Session 崩溃问题
- **问题**: 在 ACP 环境中嵌套启动 Claude Code CLI session 会触发 "Claude Code cannot be launched inside another Claude Code session" 错误。
- **修复**: 在启动 ACP agent 进程（`acp.stdio.spawn_agent_process`）和探测 ACP 服务（`acp.sync_adapter`）时，显式从环境变量中移除 `CLAUDECODE`，确保子进程环境隔离。

### 4. 测试环境兼容性
- **问题**: Python 3.12 废弃了 `asyncio.get_event_loop()` 的隐式创建行为，导致测试失败。
- **修复**: 更新测试 helper，在没有当前 loop 时显式创建新的 event loop。

## 下一步建议
- 持续监控 Spec 模式的长程任务稳定性，特别是 compact state 机制在高频保存下的性能表现。
- 考虑为 Spec 模式增加更完善的断点续传测试用例，覆盖更多边缘情况（如网络中断、进程被杀等）。
