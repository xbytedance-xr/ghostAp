## Renderer 迁移 + BaseHandler API 统一（B-016-6 / B-016-12）

### 任务描述
解决 Backlog B-016-6（三 Renderer 从 EngineCardSender 迁移到 DirectCardSession）和 B-016-12（BaseHandler 旧 API 方法迁移），实现卡片发送管线统一。

### 执行内容

#### Phase 1: Renderer 迁移到 DirectCardSession
- 新增 `_StreamThrottle` 类到 `renderers/base.py`，提取自 EngineCardSender 的节流逻辑
- 新增 `_create_direct_session` 工厂函数，创建 DirectCardSession 实例
- 更新 conftest.py mock fixture（`_mock_direct_session`）桥接 handler 方法
- 迁移 DeepRenderer / LoopRenderer / SpecRenderer 从 EngineCardSender 到 DirectCardSession
- 多消息模式通过 `session.close()` + 重建实现
- 删除 `_create_engine_sender` 旧工厂

#### Phase 2: BaseHandler 新 API 方法
- 新增 5 个 BaseHandler 方法：`reply_text`, `reply_card`, `update_card`, `send_card_to_chat`, `send_text_to_chat`
- 新增辅助方法：`_resolve_origin`, `_link_reply_response`
- 迁移 4 个 handler 文件的调用点：
  - `programming.py`: 6x `reply_message_with_id` → `reply_card`, 14x → `reply_text`
  - `project.py`: `reply_message_with_id`→`reply_card`, `reply_message`→`reply_text`, `patch_message`→`update_card`
  - `system.py`: 11x `reply_card`, 7x `reply_text`, 1x `update_card`
  - `lock_commands.py`: 6x `reply_card`, 6x `reply_text`, 2x `send_card_to_chat`, 3x `send_text_to_chat`
- 更新 `send_error_card` 成功路径使用 `reply_card`/`send_card_to_chat`，fallback 用 `reply_text`/`send_text_to_chat`
- ws_client `send_lock_conflict_card` fallback 恢复（使用 `_reply_message`）
- 旧方法（`reply_message`, `patch_message`, `send_message`, `reply_message_with_id`）保留供基础设施（action_registry, lock_helper, retry_handler 等）继续使用

#### 测试修复
- `test_loop_patch.py`: 重写为 DirectCardSession 行为测试
- `test_renderer_throttling.py`: unittest→pytest 转换 + enum 修复
- `test_spec_interaction.py`: 捕获机制从 `send_message` 改为 `reply_card`
- `test_handlers.py`: 多处断言更新
- `test_empty_error_guard.py`: 适配新 API（`reply_card`/`reply_text` 捕获）
- `test_ws_client_patch.py`: fallback 测试从 `reply_message` → `_reply_message`
- 修复 lint-style 测试：bare `e` → `str(e)`，bare `f"{e}"` → `f"{str(e) or '...'}"` 

### 技术要点
- **DirectCardSession 替代 EngineCardSender**：轻量级入口，委托 CardDelivery.deliver() 处理 create/update 语义
- **_StreamThrottle**：独立节流类，min_interval + min_new_chars + plan throttle
- **新旧 API 并存策略**：新方法（reply_text/reply_card 等）与旧方法（reply_message/patch_message 等）共存，旧方法标记 deprecated 但保留供未迁移的基础设施使用
- **旧方法完全删除 deferred**：action_registry, lock_helper, retry_handler, ws_client 等仍依赖旧 API，需要单独的基础设施迁移 pass
