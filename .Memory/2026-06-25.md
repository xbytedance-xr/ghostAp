## Workflow 确认卡 note 标签修复

### 背景 / 症状
- 用户反馈 `/wf` 仍无法工作，选完主编排 Agent 和评审 Agent 后，界面停留在“Workflow — 生成脚本中...”
  加载卡片。
- 最近 `logs.log` 显示脚本生成阶段已经进入 Traex ACP session，并在约 72 秒后关闭；随后确认卡更新和 fallback
  发送都被飞书拒绝：
  `ErrCode: 200861; ErrPath: ROOT -> body -> elements -> [8](tag: note); ErrMsg: cards of schema V2 no longer support this capability; ErrorValue: unsupported tag note`。
- 因确认卡 patch/send 连续失败，用户只看到旧的“生成脚本中”卡片，误表现为 Workflow 卡住。

### 改动
- `src/feishu/handlers/workflow.py`：
  - 将确认卡中的 fallback 警告、工具不匹配提示和空子 Workflow 引用提示从 `tag: note` 改为
    Schema 2.0 支持的 `markdown`。
  - 保留原文案和首屏提示位置，不改 Workflow 状态机、工具选择或脚本生成逻辑。
- `src/workflow_engine/selection_flow.py`：
  - 将 Workflow 选择流里的 inline error 从 `note` 改为 `markdown`，避免错误提示卡同样被飞书 V2 拒绝。
- 回归测试：
  - `tests/test_workflow_renderer.py` 新增确认卡禁止 `note` 的覆盖，包含正常、fallback、工具不匹配、空子 Workflow
    引用路径。
  - `tests/test_workflow_selection_controller.py` 新增选择卡错误提示禁止 `note` 的覆盖。
