# 2026-04-18 项目记录

## /simplify 续做：LLM 缓存复用、渲染器收口与项目持久化一致性

### 任务描述
继续按 `/simplify` 对前一轮未提交改动做彻底清理，重点处理复用/效率审查中剩余的真实问题，避免引入大规模架构重写。

### 执行内容
1. **抽取共享 LLM 缓存工具**
   - 新增 `src/utils/llm.py`，统一封装 `get_cached_chat_openai()`
   - 缓存 key 覆盖 `base_url`、`model`、`temperature` 和 `api_key` 哈希，避免仅按 temperature 命中旧 client
   - 增加全局锁与 `maxsize=8` LRU 风格淘汰，避免并发下重复构造和无界增长
2. **Loop / Spec / Intent 接入共享缓存**
   - `src/spec_engine/criteria.py`、`src/spec_engine/review.py` 改为复用共享 helper
   - `src/loop_engine/engine.py`、`src/spec_engine/engine.py` 的实例缓存改为统一 key 结构，不再各自复制构造逻辑
   - `src/agent/intent_recognizer.py` 也改为复用共享 helper，统一 Ark `ChatOpenAI` 构造与缓存策略
3. **ACPEventRenderer 收口为脏标记重建**
   - `src/acp/renderer.py` 保留 `_text_chunks` 作为可原位替换的 SSOT
   - 新增 `_text_dirty`，仅在聚合工具行被替换时延迟 `"".join(...)` 重建 `_text_content`
   - 已完成工具不再保留完整 `_completed_tools` 列表，仅保留 `_completed_tool_count`，减少长会话内存占用
4. **ProjectManager 持久化一致性补齐**
   - `src/project/manager.py` 中 `get_or_create_project_for_path()` 命中已有项目且无 `chat_id` 时，`touch()` 后立即 `_save_projects()`，避免重启后丢失最近访问时间
   - `set_active_project()` 中同项目重复激活仍走快速返回，但会在 `touch()` 后持久化，保证 `last_active` 与磁盘一致
   - 新增 `tests/test_project.py` 两个回归用例覆盖上述场景

### 技术要点
- 新增共享 helper 时保留 `llm_cls` 注入，兼容现有单测对 `ChatOpenAI` 的 patch 路径
- `api_key` 不直接进入缓存 key，改存哈希，减少明文 secret 在进程内长期悬挂
- `ACPEventRenderer` 现在只在脏时重建文本，普通 `TEXT_CHUNK` 追加仍保持 O(1)
- `ProjectManager` 本轮不引入 debounce/background flush，只修正持久化语义不一致问题，保持行为可预测
- 本轮未处理 `loop_engine/engine.py` 中更大范围的 review/criteria 逻辑复用，因涉及现有测试接口与更大改动面，先止于低风险高收益收口
