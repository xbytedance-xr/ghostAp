## 飞书卡片 schema 2.0 根级 `elements` 发送失败修复
### 任务描述
修复飞书回复消息创建 interactive card 时出现 `ErrCode: 200621`、`unknown property, property: elements, path:` 的问题，确保 schema 2.0 卡片发送给飞书时不会携带非法根级 `elements` 字段。

### 执行内容
- 在 `src/feishu/handlers/base.py` 增加 `_normalize_interactive_card_content()`，对 outgoing interactive payload 做统一规范化。
- 对 schema 2.0 卡片：若同时存在根级 `elements` 与 `body.elements`，发送前自动移除非法根级 `elements`；若只有根级 `elements`，则先下沉到 `body.elements` 再移除。
- 将该规范化逻辑接入 `_inject_ref_note()` 前置阶段，使所有通过 `BaseHandler.reply_message()/send_message()` 发出的 interactive card 共用同一修复路径，而不是在单个卡片 builder 上打补丁。
- 补充 `tests/test_handlers.py` 回归测试，验证 schema 2.0 payload 会在发送前被规整为飞书可接受结构。

### 技术要点
- 根因不是重试逻辑或飞书 SDK，而是内部同时维护了“测试/兼容读取方便”的根级 `elements` 与 schema 2.0 正式 `body.elements`；飞书服务端只接受后者。
- 选择在发送层统一规范化，而不是逐个 builder 清理，一次性覆盖所有 `BaseHandler` interactive 回复路径，避免同类错误在其他卡片构建入口重复出现。

## ACP 统一入口交互实现（/acp 工具/模型选择）
### 任务描述
检查 aiden/coco/codex/claude 的 ACP 接入与模型可用性能力，新增类似 TTADK 的 `/acp` 交互：先选工具、再选模型，然后进入对应编程模式。

### 执行内容
- 在系统命令中新增 `/acp` 入口与动作链路：`select_acp_tool`、`select_acp_model`、`refresh_acp_models`。
- 新增 ACP 工具/模型选择卡片：
  - `CardBuilder.build_acp_tool_select_card()`
  - `CardBuilder.build_acp_model_select_card()`
- 新增 ACP 模型实时探测：通过 `spawn_agent_process + initialize + new_session` 读取 `available_models`；失败时为 coco 提供 `coco_model_manager` 回退。
- 新增快捷菜单入口 `show_acp_menu`，并在 ws_client 中注册动作与 forwarding 映射。
- 为 Coco/Claude/Aiden/Codex 模式处理器增加 `current_model` 覆盖能力，使 `/acp` 选择的模型可透传到会话启动。
- 在 `ProjectContext` 增加 `acp_tool_name/acp_model_name` 并加入快照序列化，支持项目级记忆。

### 技术要点
- 模型列表优先使用 ACP 协议返回（`new_session.models.available_models`），满足“协议实时获取”目标。
- 交互复用了 TTADK 的卡片/动作机制，避免新增并行 UI 框架。
- 进入执行阶段仍复用既有编程模式 handler（coco/claude/aiden/codex），降低改动面。
