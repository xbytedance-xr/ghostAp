# 2026-05-26

## Shell 默认改为空黑名单

### 背景

用户希望远程 Shell 不再被白名单和硬编码危险模式过度拦截，默认可以自由执行 Shell；如需限制，只通过黑名单配置显式拦截。

### 变更

- `Settings` 默认 `sandbox_command_blacklist=""`、`sandbox_use_whitelist=False`，源代码默认不再带危险命令列表。
- `SandboxExecutor` 默认策略链移除 `DangerousPatternCheckStrategy`，只保留显式 opt-in 的白名单策略和配置黑名单策略；空黑名单不会拦截 `&&`、管道等普通 shell 组合。
- `BlacklistCheckStrategy` 不再因为白名单开启而跳过，显式配置的黑名单始终生效。
- `.env.example` 删除旧白名单示例，新增空 `SANDBOX_COMMAND_BLACKLIST=` 示例。
- `tests/test_sandbox.py` 更新为新契约：默认空黑名单放行复合 Shell；破坏性命令测试显式配置黑名单，避免误执行真实危险命令。

### 验证

- 红灯确认：新增默认空黑名单/复合 Shell 放行/配置黑名单拒绝 3 个测试，在旧实现下失败。
- `uv run python -m pytest tests/test_sandbox.py::TestSandboxExecutorBlacklist::test_default_settings_use_empty_blacklist_mode tests/test_sandbox.py::TestSandboxExecutorBlacklist::test_default_executor_allows_compound_shell_commands tests/test_sandbox.py::TestSandboxExecutorBlacklist::test_configured_blacklist_rejects_matching_command_text -q` → 3 passed。
- `uv run python -m pytest tests/test_sandbox.py tests/test_sandbox_security.py -q` → 45 passed。
- `uv run python -m pytest tests/test_timeout_e2e.py::TestSandboxExecutorE2E tests/test_repo_lock.py::TestIsP2PThreadLocalPropagation::test_sandbox_executor_no_independent_lock -q` → 3 passed。
- `uv run ruff check src/config/settings.py src/sandbox/executor.py tests/test_sandbox.py` → passed。
- `uv run python -m src.main --validate` → 配置校验通过。
- `git diff --check` → passed。

### 风险

- 默认 Shell 执行现在基本放行，安全边界转移到操作者信任、repo/chat lock、超时、输出截断和可选黑名单。若后续需要生产环境保护，应通过 `SANDBOX_COMMAND_BLACKLIST` 显式配置片段。

## Slock 模式集中报错修复

### 背景

用户要求修复 Slock 模式相关报错并远程重启服务。起始全量 Slock 测试曾有大量失败，主要集中在卡片 schema/移动端布局、讨论持久化、内存并发、任务分配、升级恢复、安全脱敏、零人工交互和兼容构造参数。

### 变更

- 修复 Slock 卡片构建兼容性：按钮紧凑样式、命令面板 wide screen、状态颜色、讨论详情 note 渲染、结论冲突/超时提示、escalation resolved 内容结构、任务预览截断。
- 修复 Slock 讨论流程：动态 uncertainty 触发支持 `unsure` 运行时别名、无 engine/agent 时可返回 `(None, 0)`、绑定任务上下文注入、Final Arbiter 非注册参与者保护、结论三路持久化和 Key Knowledge `[DECISION]` 同步。
- 修复 Slock 内存/路由并发：MemoryManager 增加异步写、OCC 合并、L1 容量裁剪保护、审计日志异步写、TaskClaim snapshot 和 force-assign audit。
- 修复 handler/engine 兼容性：`/task assign` 正式分发到 `assign_task`，NLI timeout/UNKNOWN 回落到 smart routing，mock 路径清理未调度 coroutine，escalation manager 支持旧构造参数，agent 执行缺失 progress/autonomous resolver 时不崩溃。
- 修复安全与脱敏：OpenAI-style key 脱敏、安全错误文案不泄露内部路径/host/秒数，ACP tool filter 缺失时显式 degraded。
- 保留用户侧更优契约：任务看板继续使用移动端安全的 `flex_mode="none"`，`/task assign` 继续可用并支持 quoted multi-word 解析。

- 全量 Slock 仍有上述 3 个规格冲突测试未通过；如果产品决定回滚旧契约，需要同步修改移动端/quoted parsing/E2E 测试或产品行为，不能只改实现。
- 全量测试仍有若干历史 warning，主要来自测试 mock 的 async/NLI 和 bootstrap 路径；当前未影响通过率。
