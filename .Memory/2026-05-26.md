# 2026-05-26

## Shell 默认改为空黑名单

### 背景

用户希望远程 Shell 不再被白名单和硬编码危险模式过度拦截，默认可以自由执行 Shell；如需限制，只通过黑名单配置显式拦截。

### 变更

- `Settings` 默认 `sandbox_command_blacklist=""`、`sandbox_use_whitelist=False`，源代码默认不再带危险命令列表。
- `SandboxExecutor` 默认策略链移除 `DangerousPatternCheckStrategy`，只保留显式 opt-in 的白名单策略和配置黑名单策略；空黑名单不会拦截 `&&`、管道等普通 shell 组合。
- `BlacklistCheckStrategy` 不再因为白名单开启而跳过，显式配置的黑名单始终生效。
- `.env.example` 删除旧白名单示例，新增空 `SANDBOX_COMMAND_BLACKLIST=` 示例。
- `tests/test_sandbox.py` 更新为新契约：默认空黑名单放行复合 Shell；破坏性命令测试显式配置黑名单，避免误执行真实危险命令。

### 风险

- 默认 Shell 执行现在基本放行，安全边界转移到操作者信任、repo/chat lock、超时、输出截断和可选黑名单。若后续需要生产环境保护，应通过 `SANDBOX_COMMAND_BLACKLIST` 显式配置片段。
