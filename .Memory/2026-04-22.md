## 并发风险排查与锁竞争优化
### 任务描述
排查并发风险：优化 `src/acp/manager.py` 和 `sync_adapter.py` 中的锁竞争与 `event.wait()`，确保完备的超时与状态恢复机制。

### 执行内容
1. **解决 ACP 异步操作的并发泄漏**：
   - 在 `sync_adapter.py` 的 `_run_async` 方法中，当 `future.result(timeout=timeout)` 抛出 `TimeoutError` 时，主动调用 `future.cancel()`。
   - 这防止了因为客户端同步线程超时退出后，后台 asyncio event loop 仍在继续执行该协程（导致资源泄漏和状态不一致）的问题。
2. **解决 ACP Session Manager 的锁竞争问题**：
   - `manager.py` 的 `_end_session_unlocked` 方法中，原来是在持有全局 `self._lock` 的情况下同步调用 `session.close()`。
   - `session.close()` 内部涉及 asyncio 线程的安全关闭和子进程的结束，可能阻塞长达 5 秒。这会导致期间所有其他的 Session 启动、心跳、查询操作都被阻塞（典型的锁竞争/饥饿问题）。
   - 将 `session.close()` 的调用重构为在后台 daemon 线程中异步执行（`threading.Thread(target=_close_bg, daemon=True).start()`），从而立即释放全局锁，大幅提升了并发处理能力。
3. **修复测试覆盖与依赖注入带来的 Mock 失效**：
   - 之前的依赖注入重构将 `create_engine_session` 改为通过构造函数注入（`_session_starter` 和 `_create_session_fn`）。
   - 这导致了原本在 `test_engine_timeout.py` 和 `test_timeout_e2e.py` 中通过 `@patch` 直接替换模块级函数的测试失效（测试实际上调用了真实的 LLM 发起网络请求）。
   - 修复方案：将 `@patch` 替换为在测试初始化时直接对实例赋值 `loop_engine._create_session_fn = lambda **kwargs: _TimeoutSession()`。测试恢复正常并通过。

### 技术要点
- **并发协程清理**：`asyncio.run_coroutine_threadsafe` 返回的 `concurrent.futures.Future` 在抛出 `TimeoutError` 时不会自动取消底层的 asyncio task，必须显式调用 `cancel()`。
- **锁粒度与 I/O 隔离**：在持有全局锁的临界区内绝不能执行耗时的 I/O 操作或阻塞等待（如关闭子进程、等待 Event/Future）。将资源释放卸载到后台线程是解决此类锁竞争的标准模式。
- **DI 测试最佳实践**：使用依赖注入（Constructor Injection）后，Mock 测试应当顺应设计，直接注入 Mock 对象，而不是继续使用 `unittest.mock.patch` 去拦截模块的导入路径。

## ACPEventRenderer reset()/get_final_content() 语义一致性修复
### 任务描述
解决 `ACPEventRenderer.reset()` 使用 `.clear()` 原地修改可变容器（`_text_chunks`/`_active_tools`/`_modified_files`）与 `__init__` 创建新对象语义不一致的问题。外部调用者在 `reset()` 前获取的引用会被静默清空，导致难以排查的 side-effect，且 docstring 声明"外部引用不会被 retroactively updated"与 `.clear()` 的实际行为矛盾。

### 执行内容
1. **修改 `reset()` 方法**：将 `self._text_chunks.clear()` → `self._text_chunks = []`、`self._active_tools.clear()` → `self._active_tools = {}`、`self._modified_files.clear()` → `self._modified_files = set()`，与 `__init__` 语义完全一致。
2. **修改 `get_final_content()` 方法**：将 `self._active_tools.clear()` → `self._active_tools = {}`，消除同类隐式共享风险。
3. **更新 `reset()` docstring**：修正外部引用行为描述，从"will not be retroactively updated"改为"remain independent after reset() — they still hold the data captured before the reset"，文档与实现一致。
4. **新增 2 个引用独立性测试**：
   - `test_reset_does_not_mutate_prior_references`：验证 reset() 后旧 list/dict/set 引用内容不变、renderer 自身容器为新空实例。
   - `test_get_final_content_does_not_mutate_prior_active_tools_reference`：验证 get_final_content() 后旧 _active_tools 引用内容不变。

### 技术要点
- **`.clear()` vs `= []` 语义差异**：`.clear()` 原地修改容器（所有引用共享同一对象），`= []` 创建新对象并重绑定（旧引用独立）。对于 `reset()` 这种"恢复到初始状态"的语义，赋值新实例更安全、更符合直觉。
- **全局搜索确认无依赖**：`src/` 中无任何代码持有 renderer 内部容器的外部引用，所有调用点仅通过 `reset()` / `get_final_content()` / `process_event()` 等公开方法交互，修改无风险。

## 落实改进建议：审查异常 futures unfinished 纵深清洗
### 任务描述
解决审查异常中 stdlib `concurrent.futures` 内部诊断信息 `"N (of M) futures unfinished"` 泄露到用户可见 `suggestions` 的问题，在 `errors.py` 源头层和 `perspective_worker.py` 调用层双重加固。

### 执行内容
1. **errors.py 源头层纵深防御**：在 `fmt_error()` 的 TimeoutError 分支中，添加预编译正则 `_FUTURES_UNFINISHED_RE`（`r"\d+\s*\(of\s*\d+\)\s*futures?\s*unfinished"`），对 `str(exc)` 进行清洗后再判断是否为空——若整个消息仅为 stdlib 内部格式则回退到友好文案 `"操作超时，请稍后重试"`，若混有其他内容则仅清除 futures 部分保留剩余。同时覆盖 `_has_timeout_in_chain` 分支的 `inner_msg` 路径。
2. **PerspectiveWorker.run() 调用层收敛**：当 `err_code == ReviewErrorCode.TIMEOUT` 时，将 `err` 从 `get_error_detail(e)` 的原始输出强制覆盖为 `"当前系统较繁忙，操作已超时"`，确保 `suggestions=[f"审查异常：{err}"]` 中不含技术术语。
3. **run_workers_parallel() 内层 except 收敛**：同上，当内层 `fut.result()` 抛出的异常被判定为 TIMEOUT 时，`err` 同样覆盖为域语义文案。
4. **新增 8 个测试**：`test_error_formatting.py` 6 个（`TestFuturesUnfinishedSanitization` 类：纯 futures 消息/混合消息/多数量/自定义消息保留/get_error_detail 路径）+ `test_perspective_worker_timeout.py` 2 个（`PerspectiveWorker.run()` 级别清洗 + `run_workers_parallel` 内层 worker 清洗）。

### 技术要点
- **正则精确度**：`_FUTURES_UNFINISHED_RE` 仅匹配 `\d+\s*\(of\s*\d+\)\s*futures?\s*unfinished`（stdlib `concurrent.futures._base._AsCompletedWaiter` 生成的格式），不会误命中用户自定义的 TimeoutError 消息。
- **纵深防御三层**：(1) `errors.py` 源头清洗（全局兜底，所有 TimeoutError 路径受益）；(2) `PerspectiveWorker.run()` 调用层覆盖（确保 worker 级别 suggestions 干净）；(3) `run_workers_parallel()` 外层 except 已有覆盖（保持不变）。
- **日志保持原始信息**：`logger.warning` 在覆盖 `err` 之前执行，日志中仍记录 `get_error_detail(e)` 的原始输出，便于运维排查。
