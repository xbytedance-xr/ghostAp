## /new-chat 老项目路径不匹配降级查找修复
### 任务描述
用户对已有老项目执行 `/new-chat` 时，如果项目注册的 `root_path` 与当前 cwd 不一致，`find_project_by_path` 返回 None 导致流程走到 Branch C，`create_project` 检测到同名 project_id 已存在后报错"创建项目失败: 项目 xxx 已存在"。设计上应兼容老项目情况——更新资料后建群并绑定。

### 根因分析
- `find_project_by_path(path, chat_id=None)` 使用严格路径相等匹配（`ctx.root_path == expanded`）
- 老项目可能在不同目录注册，当前 cwd 与注册路径不一致时返回 None
- 此后 `create_project` 生成相同 `project_id`（name→lowercase→underscore），检测到已存在返回失败

### 修复内容
- 在 `src/project_chat/service.py` 的 `find_project_by_path` 返回 None 后，增加 `find_project_by_name(name, chat_id=None)` 降级查找
- 找到老项目后更新其 `root_path` 和 `working_dir` 为当前路径，并持久化
- 更新后正常进入 Branch A（已绑群→返回跳转卡）或 Branch B（未绑群→建群绑定）

### 技术要点
- `find_project_by_name` 传 `chat_id=None` 跳过可见性过滤（与 Hard Constraint 一致）
- 路径更新 + `_save_projects()` 确保持久化
- 新增 `TestLegacyProjectPathMismatch` 测试类（2 个用例）覆盖降级→Branch B 和降级→Branch A 两条路径
