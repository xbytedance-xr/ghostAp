## Deep Engine 模块
### 任务描述
实现复杂任务自动拆解与执行引擎。

### 执行内容
新增 `src/deep_engine/` 模块：
- `models.py` — DeepTask, DeepProject, ParsedRequirement 数据模型
- `parser.py` — RequirementParser（LLM 解析用户需求）
- `planner.py` — TaskPlanner（需求拆解为任务列表，支持依赖）
- `executor.py` — TaskExecutor（调用 Coco 执行，流式输出 + 失败重试）
- `engine.py` — DeepEngine 顶层编排器
- `reporter.py` — ProgressReporter 进度报告

命令：`/deep <需求>` 启动、`/deep_status` 查看进度、`/stop_deep` 停止

### 技术要点
- 执行流程：解析需求 → 生成任务列表 → 依次执行 → 实时进度 → 汇总报告
- 支持任务依赖关系

## Claude 编程模式全面修复
### 任务描述
修复 Claude 模式报错 `Invalid session ID. Must be a valid UUID`，全面适配卡片/项目/快照。

### 执行内容

**根因修复**：
- Session ID 从 `feishu_claude_{chat_id}_{timestamp}` 改为 `uuid.uuid4()` 生成的合法 UUID
- 第一条消息用 `--session-id`，后续用 `--resume`，不再同时传两个参数

**卡片按钮适配**：
- Claude 模式显示「退出Claude」+「切换项目」
- Smart 模式显示「Coco模式」+「Claude模式」
- 流式卡片传 `is_claude_mode=True`
- 项目创建卡片新增「开始 Claude」按钮

**项目管理兼容**：
- ProjectContext 新增 claude_mode、claude_session_snapshot 字段
- ClaudeSessionSnapshot 数据类
- 退出保存快照，下次可恢复；切换项目显示恢复卡片

**卡片动作**：
- 新增 enter_claude、exit_claude、resume_claude、new_claude 四个动作
- 进入 Claude 时自动退出 Coco（互斥保护）
- 回复编程消息自动识别 Claude/Coco 并自动进入

### 技术要点
- Claude CLI 的 session-id 必须是合法 UUID
- --session-id 和 --resume 不能同时传递
- 总测试数 214
