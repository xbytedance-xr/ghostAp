16. **`src/worktree_engine/manager.py`**: 新增 `sync_worktree()` 方法，返回 `(state, warning)`。 ✅
17. **`tests/test_worktree_sync.py`**: 3 个测试覆盖同步重置到干净状态、脏状态拒绝（非 force）。 ✅

#### AC6: 存储优化（30% 缩减目标）
18. **`src/worktree_engine/git_service.py`**: 新增 `optimize_storage()` 方法，执行 `git gc --aggressive --prune=now && git repack -a -d`；`create_units()` 批量创建后自动执行 `git gc --auto`。 ✅
19. **`src/worktree_engine/manager.py`**: `cleanup_worktrees()` 末尾集成 `optimize_storage()` 调用。 ✅
20. **`tests/test_worktree_performance.py`**: 利用 git worktree 原生共享 objects 机制验证存储共享（对象数量不随 worktree 数量线性增长）。 ✅

#### AC7: 批量创建性能（10 个 < 10s）
21. **`src/worktree_engine/git_service.py`**: 采用串行 `git worktree add` 执行（避免 `.git/worktrees` 目录锁竞争），单次创建延迟极低（~ms 级）。 ✅
22. **`tests/test_worktree_performance.py`**: 5 个测试覆盖批量创建计时、存储共享验证、独立操作验证。 ✅

#### 导出与集成
23. **`src/worktree_engine/__init__.py`**: 导出 `DeleteWarning`、`WorktreeInfo` dataclass。 ✅

### 技术要点
- **串行 vs 并行**: 采用串行 worktree add（非并行），因为 git 对 `.git/worktrees` 目录使用文件锁，并行会导致锁竞争
- **API 返回模式**: 删除安全检查使用 `DeleteWarning` 返回对象（非异常），调用方通过 `force=True` 确认，解耦 UI 与引擎层
- **Porcelain 解析**: `git worktree list --porcelain` 输出为稳定格式，适合程序化解析（相比人类可读格式）
- **单次 fetch 优化**: 远程分支关联时执行单次 `git fetch --all`，消除逐分支 fetch 的冗余网络 I/O
- **存储共享**: git worktree 原生使用共享 objects 目录，无需额外优化即可实现存储节约
- **破坏性操作安全**: `sync_worktree` 使用 `git reset --hard + git clean -fd`，默认拒绝脏状态
