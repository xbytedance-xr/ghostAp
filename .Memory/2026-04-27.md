## 审查超时韧性优化（纵深三层改进）

### 任务描述
解决「审查异常：当前系统较繁忙，操作已超时」频繁触发问题。该问题产生于 Spec/Loop/Worktree 引擎的多视角审查阶段，当 ACP prompt 调用超时时统一转换为域语义文案展示给用户。

### 执行内容

**源头层——配置调优（减少误触发）：**
- `spec_review_timeout` / `loop_review_timeout`: 120 → 180（+50%）
- `spec_review_min_timeout` / `loop_review_min_timeout`: 30 → 45
- `spec_review_hard_floor` / `loop_review_hard_floor`: 15 → 20
- `spec_review_failure_max_consecutive`: 3 → 4（更高容忍度）
- `spec_review_failure_cooldown_cycles`: 3 → 2（更短恢复时间）

**韧性层——自适应超时 + 重试强化：**
- `compute_adaptive_timeout` 衰减因子从 2**n 改为 1.5**n（更平缓曲线）
  - 旧：120 → 60 → 30 → 30（floored）
  - 新：180 → 120 → 80 → 53 → 45（floored）
- PerspectiveWorker `RetryPolicy`: max_retries 1→2, retry_delay 2.0→1.5

**用户体验层——文案增强：**
- 超时文案统一更新为：`当前系统较繁忙，操作已超时。建议：稍后自动重试，或通过 /spec resume 手动恢复`
- 覆盖 perspective_worker.py（3 处）、review.py（1 处）、dispatcher.py（1 处）

### 技术要点
- 衰减因子从 `base // 2**n` 改为 `int(base / 1.5**n)`，使用浮点除法 + int 截断
- 1.5x 衰减在 n=4 时才触底（vs 旧 2x 在 n=2 即触底），给予系统更多恢复窗口
- 熔断参数调整策略：提高阈值（4 次连续失败才熔断）但缩短冷却（2 cycle 即恢复），平衡可用性和保护性

## 审查超时韧性优化（第二轮：并发度+闭环+可观测性）

### 任务描述
在第一轮韧性优化基础上，进一步落实改进建议：提升并发度减少排队超时、强化全量超时后的自动恢复闭环、增加结构化日志便于根因定位。

### 执行内容

**并发度提升——减少排队超时概率：**
- `spec_review_max_parallel`: 2 → 3（5视角排队从3批降至2批）
- `_conduct_review_pipeline` budget 公式冗余系数：`multiplier+1` → `multiplier+2`

**自动恢复闭环——全量超时信号传播：**
- `_conduct_review_pipeline` else 分支增加全量超时检测（`all_timeout`）
- 全量超时时递增 `circuit.consecutive_timeouts` 和 `circuit.review_failure_consecutive`
- 使下一轮自适应超时衰减自动生效，避免重复用同一个过长 timeout 再次全量超时

**可观测性提升——结构化日志：**
- `PerspectiveWorker.run()` 超时日志增加 `elapsed_ms` 和 `configured_timeout` 字段
- `run_workers_parallel` TimeoutError 路径增加汇总日志：未完成 worker 数 + per_worker_timeout

### 技术要点
- 全量超时判定：`all(o.error_code == TIMEOUT for o in failed_workers)`，仅当所有失败 worker 均为 TIMEOUT 时才递增，部分超时走原有 partial_failure 路径
- budget 计算：max_parallel=3 → multiplier=ceil(5/3)=2 → budget=180×max(2,2+2)=720s
- 日志字段 `elapsed_ms` 用 `time.monotonic()` 差值计算，精度到 ms

- `src/config.py` — spec_review_max_parallel 2→3
- `src/spec_engine/review.py` — budget 公式 + 全量超时 circuit 递增
- `src/spec_engine/perspective_worker.py` — Worker 日志字段 + 汇总日志
- `tests/test_config.py` — max_parallel 默认值断言
- `tests/test_review_timeout.py` — 全量超时 circuit 递增 + budget 计算验证
- `tests/test_perspective_worker_timeout.py` — 日志字段验证 + 汇总日志验证
