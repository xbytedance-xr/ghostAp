## Spec 指令机制优化：系统指令优先级 + 卡片交互 + BUG 修复

### 任务描述
优化 Spec 模式下的指令优先级体系、卡片交互流畅度，修复 resume 方法中的严重 BUG。

### 执行内容
1. **系统指令门控精细化**：新增 `_READONLY_CARD_ACTIONS` 集合（18 个只读动作），重组 `_handle_card_action` 中 gate check 逻辑顺序，只读展示类卡片动作（展开/折叠/模式切换）不受 `/help`、`/exit` 处理期间的门控阻塞
2. **引擎控制动作优先级提升**：在 `_is_system_card_action` 中添加 9 个引擎控制动作（pause/stop/resume × deep/loop/spec），获得 HIGH 优先级和独立系统线程池调度
3. **修复 resume max_cycles 零轮执行 BUG**：将 `max_cycles` 从绝对值改为相对追加（`max_cycles = start_cycle + additional_cycles - 1`），确保 resume 后总能获得额外循环
4. **修复 resume 终止处理不一致**：抽取 `_handle_max_cycles_termination()` 方法，统一 execute/resume 的三分支终止逻辑（核心标准满足→PAUSED、infinite_mode→PAUSED、其他→ABORTED）
5. **代码质量清理**：清除 execute() 中双重 `from ..utils.errors import get_error_detail` 和双重 `import gc` + `gc.collect()`

### 技术要点
- 系统指令在调度层已有四重保障（队列隔离、独立线程池、项目串行豁免、HIGH 优先级），无需修改调度器
- 卡片动作的 gate check 需在 action_type_preview 提取之后执行（代码重排序）
- resume 的 max_cycles 应基于 start_cycle 相对计算，而非使用全局绝对值
- `test_card_deduplication::test_cache_cleanup` 为预存在 flaky 测试（git stash 验证原代码也失败），与本次变更无关

## 渲染器重复代码合并（base/deep/loop/spec_renderer）

### 任务描述
合并 deep_renderer.py、loop_renderer.py、spec_renderer.py 中与 base.py 重复的代码，消除冗余逻辑。

### 执行内容
1. **提取 warning banner 到 BaseRenderer** — 新增 `_check_warning_banner(duration, is_executing)` 方法，替换 deep_renderer(3处)、loop_renderer(2处)、spec_renderer(2处) 共 7 处内联 warning banner 计算块。spec_renderer 的不同文案（"点击下方【停止】按钮后重试"）也统一为标准文案
2. **升级 BaseRenderer.get_default_ui_state()** — 扩展为包含 compact/expanded/expand_ac/view_mode/view_context 5 个字段的完整默认状态。deep_renderer 和 spec_renderer 改为 `return super().get_default_ui_state()`，loop_renderer 在 super() 基础上添加 `history_page=0`
3. **修复 spec_renderer 使用 reporter._make_progress_bar** — 将 on_review_done 和 on_project_done 中的 `reporter._make_progress_bar()` (text bar) 替换为 `self._generate_progress_bar()` (emoji bar)，与其他所有渲染器保持一致
4. **删除 loop_renderer._patch_or_send 覆写** — 该覆写缺少 base class 的 `_check_and_truncate_payload` 截断保护，且 `throttle=False` 是 `patch_message` 的默认值，行为等价。删除后 loop_renderer 自动继承 BaseRenderer 的安全截断逻辑

### 技术要点
- `reporter._make_progress_bar` 输出 text bar（`[█████░░░░░] 50%`），`self._generate_progress_bar` 输出 emoji bar（`✅✅⬜️`），两者不同，spec_renderer 应使用后者保持一致性
- loop_renderer 覆写的 `_patch_or_send` 硬编码 `throttle=False`，但 `BaseHandler.patch_message` 的默认值就是 `throttle=False`，删除覆写不改变行为，同时恢复了截断保护
- BaseRenderer 的 `get_default_ui_state` 中 `view_mode` 需设为 `"status"` 而非 `"full"`（三个子类原始默认值均为 `"status"`）

- `src/feishu/renderers/deep_renderer.py` — 替换 3 处 warning banner、`get_default_ui_state` 委托 super()
