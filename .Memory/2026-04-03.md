## One-Shot Pending Slot 编程模式重构

### 任务描述
重构编程模式（Coco/Claude/Aiden/Codex/Gemini/TTADK）的进入流程，实现"一次机会"模型：主对话开启编程模式后不立即创建 ACP 会话，首条编程指令自动创建飞书话题并在话题内运行会话；消费后主对话回到智能模式。Shell 命令不消费机会。

### 执行内容
1. **enter_mode 改造** (programming.py): `_thread_enabled` 分支仅设置 ModeManager 状态和项目模式标记，不创建 ACP session，回复引导卡片
2. **exit_mode 改造** (programming.py): 新增 `is_pending_slot` 检测（无 thread、无 session、thread 启用、在模式中），支持简化退出
3. **_dispatch_to_thread** (ws_client.py): 自动创建话题消息 → 在话题内 enter_mode → 检查 session → 成功/失败/异常三路径完整回退
4. **_is_one_shot_pending** (ws_client.py): 判断是否处于 pending 状态的辅助方法
5. **_process_with_intent 路由** (ws_client.py): 编程模式分支检测 pending 状态并分发到话题，`shell_fast_tracked` 保护
6. **_dispatch_empty_text** (ws_client.py): pending 状态空文本提示
7. **log_exception 导入修复** (ws_client.py): `_dispatch_to_thread` 异常处理调用了未导入的 `log_exception`

### 技术要点
- pending slot = ModeManager 状态已设 + 无 ACP session + 非话题内 + `thread_programming_enabled`
- `_dispatch_to_thread` 的三路径（成功/session 失败/异常）都执行 `exit_to_smart` + `_set_mode_on_project(False)` 回退
- `shell_fast_tracked` 设为 True 时 `is_in_programming` 置 False 跳过分发，进入 SMART 意图识别
- `threading.local()` 保证 `set_current_thread_id` 的线程隔离
