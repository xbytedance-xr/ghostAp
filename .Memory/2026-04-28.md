## 重构：Extract Method 拆解 SpecEngine._run_cycle_loop

### 任务描述
将 `src/spec_engine/engine.py` 中超长的 `_run_cycle_loop` 方法（约 397 行）通过 Extract Method 重构拆解为多个独立的阶段方法，降低圈复杂度，提升可读性。纯重构，不改变外部行为。

### 执行内容
将 `_run_cycle_loop`（约 397 行）拆解为协调器（约 85 行）+ 9 个 helper 方法：

1. **`_prepare_cycle_input`** — 确定 SPEC 阶段输入（首轮原始需求 / work_item spec 文件 / refinement 输入）
2. **`_run_spec_phase`** — SPEC 阶段：执行 spec prompt / 直接加载 spec 文件、产物持久化、work_item 标记、acceptance_criteria 合并
3. **`_run_plan_phase`** — PLAN 阶段：执行 plan prompt、产物持久化、解析 plan_artifact
4. **`_run_task_phase`** — TASK 阶段：执行 task prompt、解析任务列表、产物持久化
5. **`_run_build_phase`** — BUILD 阶段：执行 build prompt、产物持久化
6. **`_run_review_phase`** — REVIEW 阶段（条件执行）：审查编排、诊断持久化、返回 review_passed
7. **`_check_cycle_pause`** — 暂停检查：检测引擎状态、标记 cycle 失败、持久化
8. **`_handle_cycle_exception`** — 异常处理：暂停检测、错误记录、连续失败保护、session 重建
9. **`_finalize_successful_cycle`** — 成功后处理：criteria 评估、发现+spec 生成、metrics 快照、终止检查、session 重建

**关键设计决策**：
- `persist_every_phase` 放在各阶段方法内部（而非统一的 pause check），保持 SPEC 阶段 persist 在 work_item 处理前的原始时序
- 控制流耦合（break/continue）保持在 `_run_cycle_loop` 主方法中，helper 方法通过返回值传达决策
- 删除了 `_run_cycle_loop` 中不再使用的 `requirement` 局部变量

### 技术要点
- `_run_cycle_loop` 从约 397 行缩减到约 85 行（协调逻辑）
- 9 个 helper 方法总计约 310 行（包含原始逻辑一行不差）
- 方法签名保持不变，外部行为完全一致
- 3857 tests passed，零回归

## 修复 agent_session 包重构残留问题（循环导入 + 测试 patch 目标）

### 任务描述
`src/agent_session.py` 已被拆分为 `src/agent_session/` 包（7 个子模块），但存在两类残留问题：
1. 循环导入：`agent_session/__init__.py` → 子模块 → `acp.models` → `acp/__init__` → `acp/manager` → `agent_session`（部分初始化）→ `ImportError`
2. 测试 patch 目标未更新：`monkeypatch.setattr(agent_session, "get_settings", ...)` 和 `monkeypatch.setattr("src.agent_session.build_ttadk_subprocess_env", ...)` 指向 `__init__.py` 命名空间但对应属性未导出

### 执行内容

**1. 循环导入修复（`src/acp/__init__.py`）**
- 将 `from .manager import ACPSessionManager, AgentSessionManager` 从模块级导入改为 PEP 562 `__getattr__` 延迟加载
- 首次访问 `ACPSessionManager` 或 `AgentSessionManager` 时才触发 `acp/manager.py` 导入
- 缓存到 `globals()` 避免重复导入开销
- 完全向后兼容：`from src.acp import ACPSessionManager` 继续工作

**2. 测试 patch 目标修复（`tests/test_ttadk.py`）**

- `TestSyncTTADKCLISessionCmdArgs._patch_env`：`"src.agent_session.build_ttadk_subprocess_env"` → `"src.agent_session.ttadk_cli.build_ttadk_subprocess_env"`

### 技术要点
- **循环导入根因**：`acp/manager.py` 在模块级 `from ..agent_session import SyncClaudeCLISession, SyncSession, SyncTTADKCLISession`，而 `agent_session` 的所有子模块（`protocol.py`、`claude_cli.py`、`ttadk_cli.py`、`wrappers.py`、`factory.py`）都在模块级 `from ..acp.models import ACPEvent`，触发 `acp/__init__.py` 初始化
- **PEP 562 `__getattr__`**：Python 3.7+ 支持模块级 `__getattr__`，仅在属性未找到时调用，配合 `globals()` 缓存实现一次性延迟导入
- **patch 目标原则**：`unittest.mock.patch` 修改的是指定模块命名空间中的名称绑定；包拆分后同一名称在 `__init__.py` 和子模块中有独立绑定，需要 patch 到实际使用该名称的子模块
