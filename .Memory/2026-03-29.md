# 2026-03-29 项目记录

## Spec 模式卡片信息完整性优化

### 任务描述
优化 Spec 模式卡片的信息展示完整性：实现 phase 级进度指示器、移除 content 过度折叠、增强 cycle_done review 建议展示、抽取辅助方法去重，经 rloop 3 轮审查验收。

### 执行内容
1. **Phase 级进度指示器** — 实现 `on_phase_start` / `on_phase_done` 回调，在卡片中实时展示当前 phase 的执行状态与进度
2. **移除 content 过度折叠** — 去除不必要的内容折叠逻辑，确保关键执行信息在卡片中完整展示
3. **增强 cycle_done review 建议展示** — 优化 review 阶段完成后的建议内容渲染，使审查建议在卡片中更加清晰可读
4. **抽取辅助方法去重** — 提取重复的渲染逻辑为共享辅助方法，减少代码冗余
5. **新增 8 个测试** — 补充针对 phase 进度指示器、内容展示、review 建议渲染等场景的单元测试

### 技术要点
- `on_phase_start` / `on_phase_done` 回调与现有事件驱动渲染架构一致，通过 `ACPEventRenderer` 处理
- content 折叠策略调整需兼顾桌面端和移动端的卡片展示效果
- review 建议展示增强基于 Spec Engine 的 `SpecReporter` 输出结构
- 辅助方法抽取遵循 DRY 原则，提取到 BaseRenderer 或 spec_renderer 内部共享
- rloop 3 轮审查确保变更质量：功能正确性、代码风格一致性、测试覆盖率
