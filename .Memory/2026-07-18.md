## 员工群消息 Blob 生命周期竞态修复

### 背景与原因

- 真实群中 `@柳神` 后主 Bot 显示已直派，但员工没有回复；日志在 `10:30:25.386` 报 `employee Channel event failed closed: BlobMissingError`，随后 dispatch 因 `reason=content` 终止。
- 运行态证据显示 ingress blob 在 `10:30:25.294` 被移入 quarantine，约 92ms 后异步 Channel 回调才读取它。原实现把群事件投影放在 ACK 后的异步回调，而 dispatch worker 可先完成路由、终态化及 payload GC，形成确定的先回收后读取竞态。
- 深入回归还发现群账本与 ingress 共用加密 BlobStore，但 ingress 的 orphan GC 只把 ingress 投影视为 live set，因此会把已锚定的 `group.event.recorded` payload 错当孤儿隔离，造成后续上下文 `reason=content`。

### 实现

- 将员工群事件投影收敛到 dispatch worker 的串行路径，在 control、Router、Agent dispatch 和 terminal payload GC 之前完成；Channel acceptance 回调不再重复投影，只保留控制命令快速处理。
- GroupContextLedger 在发布和恢复 Journal 投影时向 ingress 存储所有者登记共用 blob 引用；commit 失败会释放预留，ingress orphan GC 将共用引用纳入 live set。
- BlobStore 新增安全 quarantine 恢复：仅接受 SHA-256 内容地址，使用 dir-fd、`O_NOFOLLOW`、regular `0600` 校验和密文内容地址校验，再原子移回 active root。启动重放群账本时可恢复历史上被误隔离的 payload。
- 新增顺序、回调单一职责、共享引用保留和 quarantine 恢复回归，覆盖未知投递结果导致的重复 dispatch。

### 验证

- 定向真实组合回归：259 passed。
- `uv run ruff check` 覆盖全部修改文件：通过；`git diff --check`：通过。
- 完整 `uv run pytest tests/autonomous/ -q`：2230 passed、1 skipped、1 个 pinned Lark SDK protobuf deprecation warning；skip 仍是未提供外部环境的真实租户 selector。
- 提交 `5e809afe` 推送至 `origin/agent/persistent-employee-runtime` 后重启服务为 PID 1551354；3 个群账本 payload 从 quarantine 经完整性校验恢复到 active，两个隔离员工 Channel worker 常驻，启动后无新增 ERROR/CRITICAL。

### 后续风险

- 本轮不会主动向真实群发送测试消息；部署重启后需由用户重新 `@员工` 验证真实飞书响应。若某条历史群账本 blob 同时不在 active root 和 quarantine，恢复会按设计 fail-close，不会绕过内容完整性校验。

## 持久员工运行时规划缺口收口

### 背景与原因

- 对 `docs/2026-07-17-persistent-employee-runtime-and-knowledge-workspace-plan.md` 的生产调用链复核发现，多项测试级实现尚未进入真实入口：旧 `/status`、多 @ 路由、collaboration provenance、Knowledge 恢复、Team 动态回合、主 Bot group ledger 以及 CLI 后端隔离。
- 真实租户仍只有一名 ACTIVE 员工且没有 1/10/50 soak、独立 QA 签名和写入授权，因此本轮只完成本地代码与 fail-close 验收门，不伪造外部证据，也不删除 rollback fallback。

### 实现

- Employee Actor session key 加入 effort、identity version 与 instruction digest；权限、身份或启动指令变化会强制回收旧会话。生产 dispatch tick 自动 sweep idle session。
- workspace source 改为 workforce/data/team/gateway 联合投影，恢复 active assignment、checkpoint、knowledge generation 和 source refs；归档员工不再被启动重建复活，同一 knowledge page 只投影 canonical latest 版本。
- Knowledge queue 在生产数据恢复后调用 `recover()`；Team Coordinator 等 ingress/router/outbox/dispatch projection 恢复完毕后再恢复。
- Team Coordinator 由模型参与 assign/review/revise/complete 各轮决策，完成前校验 durable done criteria；有独立员工时 reviewer 不能与初稿员工相同；每条员工贡献经该员工 Outbox 投递并锚定 collaboration provenance。
- 群路由使用 `decide_group_route()`：单 @ 只唤醒目标员工，多 @ 进入一个 TeamRun；飞书 SDK structured mentions 支持带空格的员工显示名。主 Bot group event 上移至 WS 统一入口，启用员工模式但 ledger 不可锚定时 fail-close；普通 direct 与 Team 的 enrichment failure 均可使用带 warning 的 canonical partial context。
- Claude CLI 与 TTADK CLI 保持 CLI transport，不伪装 ACP。新增 bubblewrap pre-spawn namespace：显式正向环境、项目按权限 RO/RW、workspace RO、`.env` 空文件覆盖、Vault/Journal 空 tmpfs、peer workspace 不挂载；未配置隔离时禁止 spawn。
- 三个核心构造器不再隐式默认 legacy mode，生产与测试均必须显式选择 actor/coordinator 或应急 rollback 值。
- `/status` 在存在持久员工时展示全员 runtime 总览，包括 assignment、mailbox、warm/cold/unavailable session、identity、knowledge、checkpoint 与 context quality；无持久员工时兼容旧 Slock 状态。修正 degraded/stopped 被误标为 cold。
- 注册真实 `employee_runtime_*` 卡片 action；release capture 精确绑定完整 environment envelope；新增真实 authority isolation gate、restart/reconnect 下限及 1/10/50 process/session 有界增长门。
- fallback 删除 selector 改为验证独立 QA Ed25519 attestation，裸 checkpoint 不再足以晋级；方案文档记录所需环境变量。
