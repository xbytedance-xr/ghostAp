# 2026-03-02 项目记录

## Coco 模型管理与 Spec 任务稳定性增强

### 任务描述
实现 Coco 模型动态管理能力，增强 Spec 任务执行稳定性，包括：
1. 运行时切换 Coco 模型
2. Spec 任务失败时自动重试和模型切换
3. 任务恢复机制

### 执行内容

#### 1. 新增 Coco 模型管理模块 (`src/coco_model/`)
- `models.py`: CocoModel, ModelListResult 数据类
- `manager.py`: CocoModelManager 实现
  - `get_models()`: 获取模型列表（5分钟缓存）
  - `get_current_model()`: 获取当前模型
  - `set_model()`: 切换模型
- `__init__.py`: 单例模式导出

#### 2. 修改 ACP 会话支持模型参数
- `config.py`: 新增 `coco_default_model`, `spec_max_retries`, `spec_model_switch_enabled` 配置
- `sync_adapter.py`: `resolve_agent_spec()` 支持 `model_name` 参数
- `agent_session.py`: 会话创建时传递模型参数

#### 3. 实现模型管理命令
- `/models`: 显示可用模型列表
- `/model`: 查看当前模型
- `/model <name>`: 切换模型
- 命令不受编程模式拦截

#### 4. Spec 任务重试机制
- `retry.py`: RetryPolicy, should_retry(), get_retry_delay()
- 可重试错误: Invalid params, timeout, rate limit 等
- 不可重试错误: 目录不存在, 权限拒绝 等
- `engine.py` 集成重试逻辑

#### 5. 模型自动切换
- 单模型重试失败后自动切换
- 跟踪已尝试模型列表
- 所有模型失败触发任务保存

#### 6. 任务恢复机制
- `task_persistence.py`: SpecTaskState, save/load/delete/list 函数
- 存储路径: `~/.ghostap/spec_tasks/<task_id>.json`
- `/spec_recover`: 列出可恢复任务
- `/spec_recover <task_id>`: 恢复指定任务

### 技术要点
- 模型列表缓存 5 分钟，避免频繁 CLI 调用
- 线程安全：Lock 保护共享状态
- 原子写入：tmp + replace 防止文件损坏
- 指数退避：重试延迟递增
