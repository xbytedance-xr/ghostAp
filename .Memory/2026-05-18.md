# 2026-05-18 架构决策：接受证据替代继续删除

## 决策背景

`simple` 分支代码精简实测 707 行（0.34%），远未达到 20% 目标。代码库 205K 行中约 23K+ 行属于六大高风险只读区，继续精简必然触及并发正确性边界、外部协议契约、飞书 schema 规范等不可安全删除的区域。

## 架构决策

**接受 blocking evidence 作为精简终止条件**，不再继续追求 20% LOC 削减比例。转而为高风险区建立契约测试门禁（contract test gate），使后续精简具备自动化安全网。

## 后续精简前置条件

任何对上述 6 大区域的精简操作，必须满足：
1. 对应 contract test gate 存在且绿灯
2. 新增的精简不破坏现有契约测试
3. 如果需要修改契约本身，必须先更新契约测试并获得 review
4. 所有 6 大区域必须满足下方「契约测试覆盖量化完成标准」

### 契约测试覆盖量化完成标准

#### 全区域 Gate 要求

6/6 区域均须拥有独立的 contract test gate 文件（`tests/test_*_contract.py`），且全量 `pytest` 绿灯。

#### 各区域最低 Case 数基线

| # | 区域 | Gate 文件 | 最低 Case 数 | 当前状态 |
|---|------|-----------|-------------|---------|
| 1 | 锁与会话契约 | `tests/test_lock_order_contract.py` | ≥10 | ✅ 14 |
| 2 | Feishu 卡片 JSON schema | `tests/test_card_schema_contract.py` | ≥10 | ✅ 12 |
| 3 | 字符串路由/反射 dispatch | `tests/test_command_registry_contract.py` | ≥40 | ✅ 49 |
| 4 | ACP 协议字段 | `tests/test_acp_protocol_contract.py` | ≥15 | ✅ 17 |
| 5 | 三引擎策略报告器 | `tests/test_reporter_contract.py`（待建） | ≥5 | ⬜ 待实现 |

#### Area 5 完成定义（UAT 级自动化）

至少 5 个 UAT 级自动化用例覆盖三引擎报告器（Spec/Deep/Worktree）的输出结构契约，验证：
- 报告器公开方法返回值的必需字段存在性
- 输出格式（Markdown/结构化 dict）的结构稳定性
- 阶段摘要的非空与类型约束

#### Area 6 完成定义（Helper 去重扫描自动化）

至少 3 个自动化用例验证 UI text helper 无跨模块重复定义（去重扫描），验证：
- `UI_TEXT` 字典中无重复 key 定义（跨文件扫描）
- `text.py` / `footer` / `lock_common` 三模块间无语义等价的重复文本常量
- 新增 helper 文案不与已有 `UI_TEXT` key 冲突

#### 变更约束

- 新增或修改任何 gate 文件后，全量 `uv run python -m pytest tests/ -q` 必须绿灯
- 基线数值为下限；超出基线的用例不受限制，但不得低于基线
