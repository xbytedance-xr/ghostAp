# 2026-04-12 项目记录

## 测试套件加速优化

### 任务描述
测试运行时间过长（225s），优化慢测试用例但不能影响覆盖率和功能稳定性。

### 执行内容
定位 top-3 慢测试并修复（未删除任何测试用例、未修改生产代码）：

1. **tests/test_coco_model.py**（~140s → 0.16s，最大收益）
   - 根因：多个测试未 mock `CocoModelManager._load_models_via_acp`，触发真实 ACP 子进程探测
   - 修复：新增 module 级 `autouse` fixture 统一 patch `_load_models_via_acp` 返回空列表
   - 显式测试 ACP 行为的用例（`test_load_models_*`）自带 `patch.object`，优先级不受影响

2. **tests/test_spec_gc.py::test_spec_engine_gc_collect**（16.8s → 0.58s）
   - 根因：SpecEngine 使用默认 `spec_max_cycles=500`，在构造时从 `src.engine_base.get_settings` 读取
   - 修复：在构造 engine 前 patch `src.engine_base.get_settings` 返回 `fast_settings(max_cycles=3)`

3. **tests/test_ttadk_strategies.py::test_force_interactive_env**（10s → <0.01s）
   - 根因：`_read_until_prompt(master=1, timeout=10)` 读伪 fd 等待 10 秒 timeout
   - 修复：追加 `patch.object(InteractiveStrategy, "_read_until_prompt", return_value="")`

### 技术要点
- `self.settings` 在 `__init__` 被固化，必须在构造前 patch，不能事后替换
- Autouse fixture 对局部 `patch.object(manager, ...)` 无侵入，显式 patch 仍生效
- PTY 相关测试需 mock 读循环本身，而非仅 mock `pty.openpty`

### 结果
- 总耗时：225s → 72s（3.1x 加速）
- 测试通过数：1837 全部通过，0 失败
- 生产代码零改动，稳定性零风险
