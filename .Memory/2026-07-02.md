## WF 嵌套 race 取消状态修复

- 背景：race() 原语在嵌套 workflow（模板内调用 race）中，loser agent 的取消状态无法向上传播到父引擎 state_manager 和进度卡片，导致 loser 一直显示为"执行中"。
- 根因：`src/workflow_engine/bridge.py` 中 `_execute_sub_workflow` 构造子 RuntimeBridge 时，只传了 `on_agent_call`/`on_phase`/`on_log`，未传 `on_agent_aborted`。子 bridge 收到 JS runtime 的 `agent_aborted` 通知后因回调为 None 被静默丢弃。
- 修复：子 RuntimeBridge 构造补充 `on_agent_aborted=self._on_agent_aborted`，使取消通知穿透嵌套层级。
- 测试：
  - 新增 `test_nested_workflow_race_abort_propagates_to_parent`（真实 Node 子进程）：验证父 bridge `on_agent_aborted` 收到子 workflow 内 race loser 的通知、slow 的 cancel_event 被 set、workflow 在 10s 内完成。
  - 红灯验证：移除修复后测试失败（aborted_labels=[]），确认测试准确捕获 bug。
  - 已有 `test_race_cancel_real_node_process`（test_workflow_bridge_transport.py）覆盖单层级 race 取消，保持绿灯。
