## 任务调度器 + Deep Engine 多后端 + 卡片 UI 优化
### 任务描述
引入线程级任务调度器替换 ThreadPoolExecutor，Deep Engine 支持双后端，卡片 UI 按引擎区分。

### 执行内容
**新增 `src/tasking/` 模块 — TaskScheduler**：
- per-chat 有序执行（默认 concurrency=1）
- 全局并发限制（默认 max_concurrent=10）
- `queue_key` 路由：长耗时任务独立队列
- 优先级（HIGH/NORMAL/LOW）、CancellationToken 协作式取消
- TaskContext.progress() 进度上报、TaskHandle cancel/wait/get_state
- 事件监听器 add_listener(callback)

**Deep Engine 支持 Claude 后端**：
- `engine_name` 参数，`_ai_session` 泛化为 Union[CocoSession, ClaudeSession]
- DeepEngineManager 维护两个 session_manager，根据 engine_name 自动选择

**卡片 UI**：
- Deep 卡片头部颜色按引擎区分（Coco=blue, Claude=purple）
- 移除重复进度条，按钮布局响应式统一
- 流式卡片按钮支持 desktop/mobile/responsive 三种策略

### 技术要点
- TaskScheduler 替换 ThreadPoolExecutor，提供更精细的调度控制
- Deep Engine 任务用 `queue_key=f"{chat_id}:deep"` 避免阻塞普通消息
- 新增 close() 方法做优雅退出
- 测试总数 302
