# 2026-03-09 项目记录

## TTADK 模型列表获取问题诊断

### 问题描述
用户反馈项目没有实现自动获取 ttadk 的模型列表。

### 诊断过程

#### 1. ttadk CLI 版本检查
- 版本: 0.3.8
- 无 `models/model` 子命令（`OfficialCLIModelsStrategy` 不可用）

#### 2. ProbeStrategy 验证
测试命令: `ttadk code -t <tool> -m INVALID_PROBE_FOR_DISCOVERY`

| 工具 | Available models | 结果 |
|------|------------------|------|
| claude | `glm-5-ttadk, kimi-k2.5, glm-4.7-ttadk, gpt-5.2-codex-ttadk, gpt-5.2-ttadk` | ✅ 成功 |
| opencode | `gpt-5.2-codex-ttadk, gpt-5.2-ttadk` | ✅ 成功 |
| codex | `gpt-5.2-codex-ttadk, gpt-5.2-ttadk, gpt-5.3-codex` | ✅ 成功 |
| coco | 空 | ❌ 失败 |
| trae | 空 | ❌ 失败 |
| cursor | 空 | ❌ 失败 |

#### 3. 根本原因
- ttadk 0.3.8 版本没有 `models` 子命令
- 某些工具（coco/trae/cursor）在 Invalid model 错误中没有返回 Available models 列表
- 当 Available models 为空时，ProbeStrategy 返回空列表
- 系统会 fallback 到 DEFAULT_MODELS（硬编码模型名，非 ttadk 真实模型名）

#### 4. 现有策略链
```
OfficialCLIModelsStrategy (不可用，ttadk 无 models 子命令)
    ↓
ProbeStrategy (部分工具失败，Available models 为空)
    ↓
StructuredSyncStrategy (需要项目 init)
    ↓
ProjectMetaModelsStrategy (需要项目 init)
    ↓
LocalConfigModelsStrategy (从配置文件提取)
    ↓
FileCacheStrategy (历史缓存文件)
    ↓
DEFAULT_MODELS (最后 fallback)
```

### 技术要点
1. `is_model_token()` 已正确过滤 ISO8601 时间戳噪声
2. PTY 重试逻辑存在但无法解决 Available models 为空的问题
3. 缓存文件中存在历史错误数据（ISO8601 时间戳被误当作模型名）

### 解决方案选项
1. **跨工具模型共享** - 从成功获取的工具推断模型列表
2. **更新 DEFAULT_MODELS** - 使用真实 ttadk 模型名（如 `gpt-5.2-ttadk`）
3. **用户配置预设** - 允许预定义工具的模型列表
