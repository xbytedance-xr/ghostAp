
## 并发安全修复（会话字典读写锁策略不一致)
### 任务描述
修复 ACPSessionManager 中 `_sessions` 字典读写锁策略不一致问题。原有代码在 `start_session()` 写 `_sessions` 时用了锁，但 `ensure_session()`/`get_session()` 存在竞态窗口。

### 执行内容
1. **`ensure_session()` 修复**：
   - 引入 `_safe_end_session(check_fn)` 内部辅助方法
   - 使用 double-check 模式 + `_end_session_unlocked` 在持锁情况下结束会话
   - 緻加 `is_server_running`/`is_server_healthy` 检查后的原子性保障

2. **`get_session()` 修复**：
   - 在持锁情况下执行 timeout 检查
   - 对于健康检查失败后的结束操作， 使用 double-check 确保会话一致性

### 技术要点
- Double-check 模式避免竞态窗口：释放锁后调用 `end_session` 期间其他线程可能修改 `_sessions`
- 使用 `_end_session_unlocked` 确保在已持锁情况下操作
