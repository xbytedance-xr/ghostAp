1. **`src/utils/review_diagnostics.py`（新建）** — 从 `spec_engine/review.py` 提取 `build_review_exception_diagnostics`、`normalize_review_diagnostics`、`format_review_exception_log_line` 及内部辅助函数为独立可复用模块；`format_review_exception_log_line` 新增 `prefix` 参数（默认 `"[Spec]"`），Loop Engine 使用 `"[Loop]"`
2. **`src/spec_engine/review.py`（修改）** — 删除本地诊断函数实现，改为从 `utils.review_diagnostics` 导入并 re-export，保持所有外部引用不变
3. **`src/loop_engine/engine.py`（修改）** — 三处改动：
   - `LoopReviewCircuitState` 新增 `last_review_failure_diag: Optional[dict] = None` 字段
   - `_conduct_review` except 分支调用 `build_review_exception_diagnostics` 生成结构化 diag dict，存储到 `circuit.last_review_failure_diag`
   - 日志改用 `format_review_exception_log_line(diag, diag_json=..., prefix="[Loop]")` 结构化格式
   - 熔断器打开时补充 `decision/review_circuit_open/open_until_iter/consecutive_failures` 到 diag
   - try 前 `circuit.last_review_failure_diag = None` 清除旧值（与 Spec 一致）
4. **`tests/test_loop_review_circuit.py`（修改）** — 新增 6 个测试：
   - `test_timeout_error_produces_diag_with_timeout_reason` — TimeoutError → fail_reason='timeout'

   - `test_non_timeout_error_produces_diag_with_exception_reason` — RuntimeError → fail_reason='exception'
   - `test_diag_stored_on_circuit_state` — diag dict 包含所有 stable keys
   - `test_diag_cleared_on_success` — 成功 review 后 diag 置 None
   - `test_circuit_open_diag_has_extra_fields` — 熔断打开时 diag 含 circuit open 相关字段

### 技术要点
- `review_diagnostics.py` 放在 `src/utils/` 下，不依赖任何 engine 模块，避免循环导入
- `safe_str` 内联为 `_safe_str`（不从 `spec_engine.artifacts` 导入），保持 utils 独立性
- `format_review_exception_log_line` 通过 `prefix` 参数区分 `[Spec]` 和 `[Loop]` 日志前缀
- `LoopReviewCircuitState.last_review_failure_diag` 使用 `Optional[dict] = None` 默认值，向后兼容
- Spec Engine 的所有测试（188 个）无任何修改即通过，证明 re-export 策略零风险

## 三引擎 execute/resume 顶层 TimeoutError 分支加固

### 任务描述

2. **`src/loop_engine/engine.py`** — 2 处修改：
   - `execute` except Exception 前新增 `except TimeoutError` 分支，`logger.warning` + `"Loop执行超时"` 文案，设置 `ABORTED` 状态

### 技术要点
