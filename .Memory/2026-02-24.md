## Deep/Loop 模式 Claude 引擎不生效修复

### 任务描述
Deep 模式 (`/deep`) 和 Loop 模式 (`/loop`) 始终使用 Coco 引擎，即使用户已进入 Claude 编程模式也无法使用 Claude 引擎。

### 根因分析
`BaseHandler.get_engine_name(chat_id)` 调用 `mode_manager.get_mode(chat_id)` 时**未传 `project_id`**。

而编程模式是在项目级别设置的（`enter_claude_mode(chat_id, project_id=project_id)`），`ModeManager.get_mode()` 的优先级是：project mode > chat mode > SMART。不传 `project_id` 时，查不到项目级 Claude 模式，回退到 chat 级 SMART → 始终返回 "Coco"。

### 执行内容
1. **`src/feishu/handlers/base.py`** — `get_engine_name()` 新增 `project_id` 参数，透传给 `mode_manager.get_mode()`
2. **`src/feishu/handlers/deep.py`** — 3处 `get_engine_name` 调用补传 `project_id`
3. **`src/feishu/handlers/loop.py`** — 2处 `get_engine_name` 调用补传 `project_id`
4. **`tests/test_handlers.py`** — 新增 `test_get_engine_name_claude_with_project` 测试用例

### 技术要点
- `ModeManager` 支持两层模式：chat-level 和 project-level，project-level 优先
- 编程模式（`/coco`、`/claude`）通过 `programming.py` 的 `_enter_mode_on_manager()` 在 **project-level** 设置
- Deep/Loop 是控制平面命令，在 `ws_client._process_with_intent()` 中优先于模式路由被拦截处理
- 修复后 Deep/Loop 引擎选择正确尊重当前项目的编程模式

## Deep 模式卡片显示空白工具条目修复

### 任务描述
Deep 模式执行过程中卡片出现大量空白内容：
- `🔧  ✅` — 空标题的已完成工具条目
- 每次 PLAN_UPDATE 卡片把全部历史已完成工具摘要堆叠在计划下面，导致卡片越来越长

### 根因分析
1. `renderer.py` 的 TOOL_CALL_DONE 处理不检查 title 是否为空，`client.py` 中 `title=update.title or ""` 导致空 title 被渲染
2. `deep.py` 的 PLAN_UPDATE 卡片使用 `rendered[:2000]`，而 `_render()` 包含 plan + active_tools + 全部 _text_chunks，随执行推进内容不断累积

### 执行内容
1. **`src/acp/renderer.py`**:
   - TOOL_CALL_DONE：空 title 不追加到 `_text_chunks`（仍记录到 `_completed_tools` 保留统计）
   - `_render_active_tools()`：空 title 的活跃工具不渲染
   - 新增 `render_plan_view()` 方法：只返回 plan + active_tools，不含 text 历史
2. **`src/feishu/handlers/deep.py`**:
   - PLAN_UPDATE 卡片改用 `renderer.render_plan_view()` 替代 `rendered`
3. **`tests/test_acp_renderer.py`**:
   - 新增 4 个测试：空标题不渲染到 text、空标题活跃工具不渲染、render_plan_view 不含 text、render_plan_view 空状态

### 技术要点
- `render_plan_view()` 与 `_render()` 的区别：前者只包含 plan checklist + active tools，后者还包含全部 text_chunks
- 空 title 工具仍计入 `_completed_tools` 和 `completed_tool_count`，保持统计准确性
