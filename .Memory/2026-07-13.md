## Thread Context Task 3：稳定快照与严格预算

### 实现与原因

- 用 `src/autonomous/context/assembler.py` 整体替换 permissive scaffold。每次 assembly
  共享一个 monotonic absolute deadline；Thread 从空 token 开始完整读取两遍，仅比较
  current 及以前的有序 revision，pair 不稳定最多重试一次，current 缺失、删除、system、
  重复或传播后仍不可见均以稳定 reason fail-close。
- Thread revision digest 覆盖 message/chat/thread/root/parent/sender/type、create/update、
  edit/delete、两类 position 与 content digest。相同 ID 的新 revision 仅在不可变 identity
  完全一致时替换；equal-revision conflict、跨 root/thread/sender 重绑定一律拒绝。
- Group recent 使用官方 `ByCreateTimeDesc` 的 bounded window，不再扫描整个群历史：收满后
  继续读完整 cutoff timestamp cohort，再按 create time、chat-global position、message ID
  确定性选择。窗口读取两遍并要求 digest 相等；活动 continuation 在成功、失败或 deadline
  竞态退出时通过新增的线性化 `reset_chat_traversal()` 清理，后续可从空 token 重启。
- ordinary Feishu system history 不进入 trusted prompt。唯一可信系统输入是非负 token
  reserve 与 SHA-256 constraints digest，非零 reserve 必须绑定 digest；snapshot hash 同时
  绑定 tenant/agent/bot/app/chat/root/current/resolved-thread 权限坐标。
- 字符与 token 双硬预算均使用保守 `ceil`。超限严格按整个 L2、整个 L1、最旧 Group、最旧
  unprotected Thread 顺序移除，绝不截断正文或 current；protected current/reserve 自身超限
  返回 `BUDGET`。watermark 保持 canonical pre-trim source，metrics/layers/hash 在裁剪后重算。
- 公开入口把输入验证、clock、scope、source 与 assembly 全部纳入异常脱敏边界。公开错误在
  离开 `except` 后重建，`__cause__`/`__context__` 均不保留上游正文、凭据或 traceback。

## Thread Context Task 5：生产组合、readiness 与恢复所有权

### 实现与原因

- `EmployeeDepartmentRuntime` 在 Journal/Vault 之后构造并拥有 canonical Data composition、
  employee-scoped `LarkEmployeeMessageSourceFactory`、动态 Projected Registry、L1/L2 reader 与
  `EmployeeContextService`，只暴露无 secret 的 Context port 给下一阶段 ingress。
- hire readiness 与 execution readiness 分离：首次雇佣不依赖尚不存在的 BotPrincipal；ACTIVE
  员工执行前必须重建同一 Journal head、校验 requester ACL、employee/channel generation、
  app/credential binding，并用员工凭据调用官方 Application v6 Get 精确验证 app_id。任何 replay、
  probe 或配置异常都返回稳定 blocker，不向调用方泄漏异常，也不 fallback 到主 Bot client。
- Journal writer 保留非重入 leaf lock，另加独立 transaction coordination `RLock`，把 Hire、Data
  和 workforce 的 projection refresh + commit 串行化，消除 sibling domain head race；并发回归
  同时发起第二次 hire 与 ExecutionAttempt，显式断言线程在期限内退出且两个投影 head 一致。
- runtime 区分 durable projection invalidation 与显式 Fire invalidation。restart、credential binding
  变化、无预热 retirement/ARCHIVED 都会定向 drain 员工 source；只有 durable replacement ACTIVE
  可自动 reactivate，显式撤销必须显式恢复。binding 状态与 source 操作用同一 RLock，阻止 refresh
  在 `/fire` 撤权后竞态 reopen。Vault rewrap 采用 invalidate/drain → rewrap → reactivate，失败保持关闭。
- shutdown 严格按 hire/context admission、in-flight Context、source clients、employee channels、
  group/data、hire service/writer、Vault 逆序执行；任一上游 close 失败会保留下游依赖并允许重试，
  保证没有 source 调用越过 Vault 生命周期。
- Employee release manifest 新增分页、root/thread binding、revision/edit/delete、deterministic trim、
  Context failure zero-dispatch 与 employee identity/main-Bot zero-send 六个显式 pending gate；缺失或
  false assertion 的 evidence 继续 fail-close，未伪造通过状态。
