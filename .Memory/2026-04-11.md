## Spec 引擎执行日志与渲染稳定性修复

用户反馈在 /spec 模式做完一轮后崩溃，日志文件只有 37 行，无 ERROR 信息，服务在 00:03:51 正常停止。但关键发现是 23:40:18 ACP session 启动后到 00:03:51 服务停止，中间 23 分钟完全没有任何 Spec 执行日志。

### 执行内容

1. **_run_phase 日志增强**
   - 添加阶段开始日志：`[Spec:%s] 循环 %d 阶段 %s 开始`
   - 添加阶段完成日志：`[Spec:%s] 循环 %d 阶段 %s 完成, 输出长度=%d`

   - 在 `_recreate_session_best_effort()` 后检查 session 是否为 None
   - 若为 None 则输出 ERROR 日志，便于追踪问题

3. **on_review_done 渲染 bug 修复**
   - 修复 `criteria_section` 未通过 `_render_collapsible_section` 处理的 bug
   - 修复 `sp` 变量潜在 NameError（当 engine.project 为 None 时）

### 技术要点
- Spec 执行阶段(SPEC/PLAN/TASK/BUILD)原本没有中间日志，导致长时间执行完全无法追踪
- `_render_collapsible_section` 是渲染折叠验收标准的关键方法，`on_review_done` 漏掉调用导致与其他回调不一致
- `sp = engine.project if (engine and engine.project) else None` 模式可避免 race condition
