## ACP 协议重构 — subprocess→结构化 agent 通信
### 任务描述
GhostAP 使用 subprocess CLI 调用（`coco -p`, `claude -p`）+ 纯文本流式输出，无法获知 agent 动作（文件读写、命令执行），无权限控制，无执行计划跟踪。需要重构为 ACP（Agent Client Protocol）—— JSON-RPC 2.0 over stdio 的结构化通信。

### 执行内容

**新增 `src/acp/`（6 文件）**：
- `models.py` — ACPEvent, ACPEventType (TEXT_CHUNK/THOUGHT_CHUNK/TOOL_CALL_START/UPDATE/DONE/PLAN_UPDATE), ToolCallInfo, PlanInfo, PlanEntryInfo, ACPSessionState, PromptResult
- `client.py` — GhostAPClient(Client): session_update() 按类型分发→ACPEvent; request_permission() 自动批准 AllowedOutcome; 文件/终端 stub
- `session.py` — ACPSession: 异步生命周期 (start/prompt/cancel/close)，使用 spawn_agent_process() + text_block()
- `sync_adapter.py` — SyncACPSession: daemon 线程 + asyncio 事件循环，run_coroutine_threadsafe() 桥接
- `manager.py` — ACPSessionManager: per-chat 会话管理，超时/清理
- `renderer.py` — ACPEventRenderer: 累积 text+tools+plan → Feishu Markdown

**重写 `src/deep_engine/`（6→3 文件）**：
- 删除 parser.py, planner.py, executor.py（agent 通过 ACP 自行规划）
- 新增 progress.py — DeepProgress 跟踪计划条目/工具调用/修改文件
- 重写 engine.py — 单次 ACP prompt 驱动整个执行; DeepEngineCallbacks 事件转发

**重写 `src/loop_engine/`（14→4 文件）**：
- 删除 12 个模块: analyzer, roles, controller, adapter, tool_integrator, optimizer, context_manager, product_analyzer, iteration_flow, task_manager, termination_detector, termination
- 新增 tracker.py — IterationTracker 处理每轮 ACP 事件
- 重写 engine.py — 单 ACP session 多轮 prompt; 收敛检测; agent 评估验收标准
- 更新 models.py — IterationRecord.iteration (was iteration_id), optional role, added focus
- 更新 reporter.py — 移除 adapter 依赖

**删除 `src/session/`（整个目录）**：
- 被 src/acp/session.py + src/acp/manager.py 替代

**更新集成文件**：
- ws_client.py — ACPSessionManager 替换 CocoSessionManager/ClaudeSessionManager
- handler_context.py — coco_manager/claude_manager 改为 ACPSessionManager 实例
- handlers/programming.py — ACPEventRenderer + on_event 回调模式
- handlers/deep.py — DeepEngineCallbacks 事件转发
- handlers/loop.py — LoopEngineCallbacks 事件转发
- config.py — 新增 acp_permission_auto_approve, acp_session_timeout

### 技术要点

**ACP SDK 关键 API**：
```python
from acp import spawn_agent_process, text_block
from acp.interfaces import Client, Agent
from acp.schema import ToolCallStart, ToolCallProgress, AgentMessageChunk, ...
AllowedOutcome(option_id=str, outcome="selected")  # 权限响应
ReadTextFileResponse(content="")  # 注意字段是 content 不是 contents
```

**架构变化**：
- subprocess CLI → JSON-RPC 2.0 over stdio
- 手动解析/规划/执行 → agent 自规划 + 事件流驱动
- 同步+异步混合 → SyncACPSession 桥接层统一

- 删除 15 个测试文件
- 更新: test_ws_client_patch.py, test_loop_models.py, test_handlers.py

## Loop Engine 移植到 multicoco 分支（不引入 ACP）
### 任务描述
multicoco 分支的 ws_client.py 已预接 Loop Engine 路由（import、_FORWARDING_MAP、intent 处理），但 src/loop_engine/ 为空、handlers/loop.py 不存在。从 acp 分支移植 Loop Engine，改造为使用 subprocess session 模块（src/session/），而非 ACP。

### 执行内容

**新建 `src/loop_engine/`（4 文件）**：
- `models.py` — 1:1 复制 ACP 分支（纯数据模型，无 ACP 依赖）
- `reporter.py` — 1:1 复制 ACP 分支（纯格式化，无 ACP 依赖）
- `engine.py` — 核心改造：SyncACPSession → CocoSessionManager/ClaudeSessionManager + BaseSession.send_prompt_streaming(); EngineRunState → _is_running/_should_stop 布尔; 删除 IterationTracker; _evaluate_criteria 用 session.send_prompt() 替代 ACP
- `__init__.py` — 导出 engine + models + reporter

**新建 `src/feishu/handlers/loop.py`**：
- 从 ACP 分支搬运，移除 ACPEvent 相关导入，删除 fmt_error 依赖（该 utils 不存在于 multicoco 分支）

**修改已有文件（6 文件）**：
- `handlers/__init__.py` — 添加 LoopHandler 导出
- `handler_context.py` — 添加 loop_engine_manager + loop_reporter 字段
- `config.py` — 添加 loop_max_iterations=10, loop_execution_timeout=7200, loop_convergence_window=3
- `intent_recognizer.py` — 添加 6 个 IntentType (ENTER_LOOP/LOOP_STATUS/STOP_LOOP/LOOP_PAUSE/LOOP_RESUME/LOOP_GUIDE)，更新 INTENT_MAP、EXACT_COMMANDS、_quick_match
- `handlers/system.py` — 添加 is_loop_command() 静态方法 + loop_handler 属性 + Loop Engine 帮助卡片
- `tests/test_handlers.py` — HandlerContext 构造增加 loop_engine_manager/loop_reporter

**新建 `tests/test_loop_engine.py`**：
- 61 个测试覆盖 LoopEngine、LoopEngineManager、Models、ContextManager、Reporter

### 技术要点
- **ACP → subprocess 适配核心**：用 `session.send_prompt_streaming(prompt, on_chunk, timeout, cwd, should_stop=...)` 替代 ACP 的 `session.send_prompt(prompt, on_event=...)`
- **LoopEngineCallbacks.on_iteration_event** 签名从 `(int, ACPEvent)` 改为 `(int, str)` 纯文本 chunk
- **LoopEngineManager** 内置 `_coco_session_manager` + `_claude_session_manager`，模仿 DeepEngineManager 模式
- **IterationTracker 被删除**：完全依赖 ACP 事件，subprocess 模式无需
- **fmt_error 不存在**：multicoco 分支无 src/utils/errors 模块，改为内联 f-string
