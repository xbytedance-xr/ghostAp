## ACP 协议重构 — subprocess→结构化 agent 通信
### 任务描述
GhostAP 使用 subprocess CLI 调用（`coco -p`, `claude -p`）+ 纯文本流式输出，无法获知 agent 动作（文件读写、命令执行），无权限控制，无执行计划跟踪。需要重构为 ACP（Agent Client Protocol）—— JSON-RPC 2.0 over stdio 的结构化通信。

### 执行内容

**新增 `src/acp/`（6 文件）**：
- `models.py` — ACPEvent, ACPEventType (TEXT_CHUNK/THOUGHT_CHUNK/TOOL_CALL_START/UPDATE/DONE/PLAN_UPDATE), ToolCallInfo, PlanInfo, PlanEntryInfo, ACPSessionState, PromptResult
- `client.py` — GhostAPClient(Client): session_update() 按类型分发→ACPEvent; request_permission() 自动批准 AllowedOutcome; 文件/终端 stub
- `session.py` — ACPSession: 异步生命周期 (start/prompt/cancel/close)，使用 spawn_agent_process() + text_block()

### 技术要点

**ACP SDK 关键 API**：
```python
from acp import spawn_agent_process, text_block
from acp.interfaces import Client, Agent
from acp.schema import ToolCallStart, ToolCallProgress, AgentMessageChunk, ...
AllowedOutcome(option_id=str, outcome="selected")  # 权限响应
ReadTextFileResponse(content="")  # 注意字段是 content 不是 contents
```

**架构变化**：
- subprocess CLI → JSON-RPC 2.0 over stdio
- 手动解析/规划/执行 → agent 自规划 + 事件流驱动
- 同步+异步混合 → SyncACPSession 桥接层统一
