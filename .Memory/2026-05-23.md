# 2026-05-23

## AI agent 产品调研报告

- 产出 `docs/ai-agent-products-research-2026-05-23.md`，系统调研 OpenClaw、Hermes 以及 Codex、Claude Code、Gemini CLI、Aider、OpenCode、Cline、Roo Code、OpenHands、Devin、GitHub Copilot cloud agent、Jules、Cursor、Windsurf、Kiro、Replit Agent 等同类产品。
- 报告按产品形态拆分为多渠道 Gateway、CLI/TUI coding agent、IDE-native agent、云端异步 PR/任务 agent、agent SDK/框架，避免把消息入口型平台和单一 coding assistant 混为一类。
- 核心结论：OpenClaw 更像多渠道个人/团队 AI Gateway，优势在消息入口、移动/语音/Canvas 和本地控制面；Hermes 更像带长期记忆和技能自改进的 CLI/TUI 开发 agent；Codex/Claude Code 更适合生产 coding；Devin/Copilot/Jules 更适合团队异步 backlog/PR；OpenHands/Cline/OpenCode/Aider 适合作为开源和 BYOK 补充。
- 对 GhostAP 的启发：借鉴 OpenClaw 的 channel/session/sandbox/tool policy 和 DM pairing，借鉴 Hermes 的长期记忆、技能沉淀、session search、多后端执行与 cron；继续保持策略轴和工具 transport 轴分离，避免把某个后端写死为产品状态。
- 验证：基于公开官方文档、GitHub README、OWASP/Snyk 安全资料和当前网页核验；本次为文档调研，无代码测试运行。
