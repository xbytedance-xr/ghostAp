# 2026-06-10 工作记录

## Claude 1M Context 支持

### 背景
trae 在 Claude Sonnet 4.5 / Opus 4.8 等模型上提供 "Max 1M context window" 选项；用户希望本项目 ACP claude provider 也具备这一能力。截图显示 trae 把 1M 模型作为模型列表里的额外条目展示，与普通模型选择体验一致。

事实锁定：当前会话本身就跑在 `claude-opus-4-8[1m]` 上（Claude Code 的 system prompt 自报），证实 `[1m]` 后缀是 Claude Code CLI 启用 1M context 的官方语法（与 ANTHROPIC_BETAS=context-1m-2025-08-07 等价）。

### 改动概览（按数据流自上而下）

**新增文件**
- `src/acp/claude_capabilities.py` — 1M capability 单一事实源：`CLAUDE_1M_PREFIXES`（`claude-sonnet-4-5/4`、`claude-opus-4-8/4-5`，前缀匹配以兼容日期化 ID）、`CONTEXT_1M_BETA="context-1m-2025-08-07"`、`SUFFIX_1M="[1m]"`，配套 4 个纯函数 `model_supports_1m / is_1m_variant / strip_1m_suffix / with_1m_suffix`。
- `tests/test_claude_1m_context.py` — 87 个用例，覆盖 capabilities helper、`apply_anthropic_betas` 行为、`_apply_model_args` 保留 `[1m]` 后缀、provider model_style 锁定、`_inject_claude_1m_variants` probe 注入逻辑。

**修改文件**
- `src/acp/providers/__init__.py` — claude `_ProviderConfig.model_style` 从 `None` 改为 `"model_long"`（**关键修复**：之前 model_name 根本没有传给 Claude CLI，所以普通模型切换也是哑操作；这次顺手修复了）。
- `src/acp/helper.py` — 新增 `_inject_claude_1m_variants(items)` 模块级函数，`probe_acp_models` 在 `tool_name=="claude"` 时调用它在原列表后追加 `<id>[1m]` 候选项；`is_default=False` 防止误成默认。
- `src/utils/env.py` — 新增 `apply_anthropic_betas(env, model_name)`：仅当 model_name 含 `[1m]` 时把 `context-1m-2025-08-07` 合并进 `env["ANTHROPIC_BETAS"]`，已有 betas 去重保序。
- `src/acp/sync_adapter.py:1577-1597` — spawn 前调用 `apply_anthropic_betas`；**修复 critical bug**：原 `ACPSession.start()` 调 `build_clean_env(base=env_override)`，base 不为 None 时会**完全替换** `os.environ` 而非合并，因此当 env_override 是 None 又需要注入 betas 时，必须先 seed 一份 `build_clean_env()`（真实 os.environ 副本），否则 spawned Claude 子进程会丢 HOME / USER / ANTHROPIC_API_KEY / OAuth tokens / locale / proxy 等关键环境，可能静默切到不同身份/计费上下文。
- `src/ttadk/models.py:838` — `ACPModelOption` 末尾追加 `supports_1m: bool = False`（向后兼容，旧 positional 调用全是 3 参数）。
- `src/card/builders/system.py:1256-1340` — `build_acp_programming_ready_card`：检测到 1M 变体时插入红色警告 banner（明示"超 200K tokens 按 2× 基础费率"+"选择会持久化、重启自动恢复"），主题色由 green 改 red，并在按钮区首位插入 danger 风格"切回标准上下文"按钮（直接 select_acp_model 到 strip_1m_suffix 后的 model id）。
- `src/card/ui_text.py` — 新增 `system_acp_1m_warning_banner` 与 `system_acp_1m_switch_standard_btn` 两个 UI_TEXT 常量。
- `tests/test_acp_provider.py:91-112` — 更新 `test_claude_provider`：原断言锁的是旧 bug（"传 model 不下发"），现在断言新行为：`--model test-model` 被注入；`[1m]` 后缀完整保留。

### 风险与回退
- 把 claude `model_style=None → "model_long"` 是行为改变，但 `_apply_model_args` 第 132 行的 `if not m: return args` 守卫保证空 model_name 路径不变。
- `[1m]` 后缀若被 Claude wrapper 吞掉，env `ANTHROPIC_BETAS` 兜底；最差退化为 200K，不会启动失败。
- UI 红色警告 + 一键切回 + 持久化提示，避免用户被坑（一次实验性勾选变成隐式默认）。
- 缓存 `_acp_probe_cache` 按 tool_name 隔离且条目确定性派生，无副作用。
