### 任务描述
为 TTADK 模式添加三项核心增强：

4. **会话保活** (`src/acp/manager.py`):

5. **测试**:

   - `tests/test_acp_keepalive.py` — 5 个测试
   - 全量 1734 passed, 10 skipped

- 自动更新使用全局 `_ttadk_update_attempted` 标志实现进程级去重
- keepalive 线程使用 `threading.Event.wait(timeout=)` 实现可中断定时

## ACPSessionManager Keepalive 后台线程
### 任务描述
为 ACPSessionManager 添加 keepalive 后台线程，定期检测空闲会话的 ACP Server 存活状态，自动清理已死亡的会话。

### 执行内容
- **Task 4.1**: 修改 `ACPSessionManager.__init__`，新增 `keepalive_interval` 和 `idle_healthcheck_s` 参数，以及 `_keepalive_stop`(Event)、`_keepalive_thread`(Thread) 属性；当 `keepalive_interval > 0` 时自动启动守护线程
- **Task 4.2**: 实现 `_keepalive_loop()` 方法：按 `_keepalive_interval` 周期轮询，快照会话列表，对空闲超过 `_idle_healthcheck_s` 的会话调用 `is_server_running()`，dead session 加锁双检后调用 `_end_session_unlocked()` 清理；整体 try/except 防止线程意外退出
- **Task 4.3**: 修改 `cleanup_all()`，在清理会话前先 `_keepalive_stop.set()` + `join(timeout=5)` 停止 keepalive 线程
- **Task 4.4**: 更新 `ws_client.py` 中全部 6 个 ACPSessionManager 实例（coco/claude/aiden/codex/gemini/ttadk），传递 `settings.acp_keepalive_interval` 和 `settings.acp_session_idle_healthcheck_s`
- **Task 4.5**: 创建 `tests/test_acp_keepalive.py`，包含 5 个测试用例：线程启动/不启动/cleanup 停止、dead session 清理、alive session 保留

### 技术要点
- 配置项 `acp_keepalive_interval=300` 和 `acp_session_idle_healthcheck_s=120.0` 已在 `config.py` 中预置
- keepalive 线程为 daemon 线程，进程退出时自动终止
- 使用 `threading.Event.wait(timeout=interval)` 代替 `time.sleep`，确保 `cleanup_all()` 能及时唤醒并退出
- 快照 + 锁外检查 + 锁内双检模式，避免持锁期间执行耗时 RPC
