## WF per-agent 超时权威化 + 进度心跳（agent_call timed out 修复）

### 背景 / 动机
用户反馈 `/wf` 执行阶段 `execute-traex` 失败：卡片停在“执行已失败”，最近变更 13:26:03。日志复盘（`logs.log`）确认真正失败点是 `Execution: agent_call timed out waiting for host response`，发生在 traex 会话启动（13:23:07）后约 **180s**——不是总超时（总超时已支持 0=不限时），而是 **JS runtime 侧单 agent RPC 看门狗**在起作用：LLM 生成的脚本给每个 `agent()` 硬编码 `timeout: 180`，`runtime.js` 的 `requestTimeoutMs('agent_call')` 就用这个小值设 181s 看门狗，真实 traex 编码任务跑过 180s 即被 abort。Python 侧 `_effective_timeout_s(min(requested, budget))` 也用脚本的 180 传给 `send_prompt`，双侧都被脚本小值卡死。用户诉求：时间不要限制、留给用户交互（停止按钮已在上一条落地）、进展及时同步、并对齐真实 env。

### 改了什么
1. **超时语义反转：host 配置为“权威下限”，脚本值只能抬高**
   - `src/workflow_engine/executor.py` `_effective_timeout_s`：`fallback`（配置值）作 floor，脚本 `requested` 只能 `max(floor, requested)`；配置 `<=0`=不限时 → 用有限 backstop（`AGENT_UNLIMITED_BACKSTOP_S`=30天）。**用 float 比较，`0.01` 等亚秒配置不被 int 截断误判为 unlimited**（fast test 依赖）。仍按剩余总预算裁剪。
   - `src/workflow_engine/bridge.py` `_cap_agent_timeout_to_remaining_budget` 同步 floor 语义；`_ensure_workflow_deadline` 额外解析 `workflow_agent_call_timeout_s` 并在 `init` 下发 `agent_call_timeout_s` 给 JS。
2. **runtime.js 看门狗解耦脚本值**
   - 新增 `workflowAgentCallTimeoutMs`（从 init 读）、`effectiveAgentTimeoutSeconds()`（floor 权威、脚本只能抬高、0→backstop）；`capTimeoutToDeadline`/`requestTimeoutMs` 改用它，不再直接信任脚本 `params.timeout`。
   - **修复 Node setTimeout 溢出 bug**：30 天 backstop(2.592e9 ms) > Node 上限 2^31-1 ms，会被静默截断成 1ms 导致看门狗“立即触发”（parallel-cap 测试 0.06s 暴露）；加 `MAX_SAFE_TIMER_MS` 钳制。
3. **配置放开 + 对齐真实 env**
   - `settings.py` `workflow_agent_call_timeout_s` 由 `ge=10` 改 `ge=0`（0=不限时；>0 为权威下限），描述更新。
   - 真实 `.env`（gitignored）补齐 WF 超时块：`WORKFLOW_TOTAL_TIMEOUT_S=0`、`WORKFLOW_AGENT_CALL_TIMEOUT_S=0`（均不限时），`.env.example` 注释同步 floor 语义。
4. **进度心跳（同步“正在干什么”）— 并行 subagent 实现**
   - `src/workflow_engine/renderer.py`：RUNNING agent 行显示 `执行中 {已运行时长}…`，顶部摘要加 `⏱ 已运行:` 行（仅非终态+RUNNING+started_at）。
   - `src/workflow_engine/engine.py`：新增 `WorkflowHeartbeat` daemon 线程，`bridge.run()` 阻塞期间每 `PROGRESS_HEARTBEAT_S`(10s) 走 coalescer 重刷进度卡，让已运行计数持续前进；`_on_stop`/`cleanup`/finally 均 best-effort 停止。
   - `src/workflow_engine/constants.py`：加 `PROGRESS_HEARTBEAT_S`、`AGENT_UNLIMITED_BACKSTOP_S`。

### 验证
- 红灯定位：`logs.log` 确认 `agent_call timed out waiting for host response` @180s 为 JS 看门狗，非总超时。
- 新增回归：executor 3（脚本小值抬到 floor 600 / 脚本大值 1800 抬高 / 0→backstop）、bridge 4（小值抬 floor、0→backstop、init 下发 floor 900）、renderer/heartbeat（已运行时长、心跳幂等/吞异常/`_on_stop`）。
